## 摘要
在时间序列生成中，现有方法通常为每个数据集手动设计或单独训练模型，这阻碍了其可扩展性，并且未能利用跨领域的共享时间结构。为解决这种碎片化问题，我们提出UPLOTS，一个面向跨领域约束时间序列生成的统一、提示引导的语言模型框架。不同于构建任务特定模型，UPLOTS利用单个预训练Transformer主干，通过习得的约束提示进行引导，实现按需生成并具备精确的模式控制。一项关键创新是我们的动态多数据集损失重加权和提示到模式的映射，这使得UPLOTS能在训练中内化多样化的时间结构，并在推理时有条件地生成它们。我们在四个真实世界基准和多种约束设置（包括高峰期、日历、负荷水平和波动性模式）上评估UPLOTS。额外的留存约束组合和下游预测实验进一步表明，UPLOTS能够泛化超越原始高峰模式设置，并在真实数据稀缺情况下改善数据增强。我们的代码和基线可在匿名GitHub仓库获取：https://anonymous.4open.science/r/UPLOTS-6C36。

## Abstract
In time-series generation, existing approaches typically handcraft ortrain a separate model for each dataset, which hinders their scalability and fails to leverage shared temporal structures across domains. To address this fragmentation, we propose UPLOTS, a Unified, Prompt-guided Language model framework fOr constrained Time-Series Generation across diverse domains. Instead of building task-specific models, UPLOTS leverages a single pre-trained transformer backbone guided by learned constraint prompts, enabling on-demand generation with precise pattern control. One key innovation is our dynamic multi-dataset loss re-weighting and prompt-to-pattern mapping, which allows UPLOTS to internalize diverse temporal structures during training and conditionally generate them at inference. We evaluate UPLOTS on four real-world benchmarks and multiple constraint settings, including peak-period, calendar, load-level, and volatility patterns. Additional held-out constraint-combination and downstream forecasting experiments further demonstrate that UPLOTS generalizes beyond the original peak-pattern setting and improves data augmentation under scarce real-data regimes. Our code and baselines are available at anonymous github repo: https://anonymous.4open.science/r/UPLOTS-6C36.

## 2. 方法论：核心思想、关键技术细节与流程

### 核心思想
- 采用预训练大语言模型（LLM）作为统一骨干，通过**时间序列提示嵌入模块（TPEM）** 将自然语言约束提示与时间序列特征融合，再结合基础时间序列生成模型（如Diffusion-TS）进行训练和生成。
- 引入**动态加权训练策略**，包括课程感知损失调节器（CALM）和滚动损失动态采样（RLDS），以在多个数据集上平衡学习进度，避免模型偏向简单或噪声数据集。

### 关键技术细节

#### 1. 时间序列提示嵌入模块（TPEM）
- 输入：时间序列 `X ∈ R^{B×F×T}`（B:批次，F:特征，T:长度）。
- 经卷积MLP投影到LLM隐藏维度：`E_ts = Conv_MLP(X) ∈ R^{B×T×D}`。
- 文本提示（如“Morning Peak Patterns in ETTh”）被分词器编码并通过LLM冻结的词嵌入层得到提示嵌入：`E_pr = WTE(Tokenizer(prompt)) ∈ R^{B×L×D}`（L:提示长度）。
- 将提示嵌入和时间序列嵌入沿时间维度拼接：`Z_0 = [E_pr; E_ts] ∈ R^{B×(L+T)×D}`。
- 输入到截断的预训练LLM（仅保留前两层Transformer，冻结除LayerNorm和位置嵌入外的全部参数）进行处理，输出再经线性映射回原始特征空间，作为基础生成模型（如Diffusion-TS）的输入。

#### 2. 动态加权训练策略（DWTS）
- **课程感知损失调节器（CALM）**：为每个数据集维护一个最近W个损失的双端队列H_i，计算运行平均ℓ̄_i。在训练早期（warm-up epochs e<5），根据各数据集平均损失动态分配静态课程权重w_i，对困难数据集赋予较小权重，以避免噪声干扰。
- **滚动损失动态采样（RLDS）**：利用各数据集损失队列计算采样概率p_i = ℓ̄_i / Σℓ̄_j，使高损失（即表现差）的数据集被更频繁地采样，从而聚焦薄弱领域。

#### 3. 基础生成模型
- 论文采用Diffusion-TS作为基本生成模型（去噪扩散），它使用Transformer预测原始信号，并同时优化时域和频域损失，最终输出分解为趋势、季节性和残差。
- TPEM输出的特征直接替代原Diffusion-TS的输入，实现条件生成。

### 算法流程（文字描述）
- **训练阶段**：将多数据集样本及其对应的自然语言提示输入TPEM，融合后送入Diffusion-TS进行联合训练。损失采用DWTS策略加权，并动态调整采样比例。
- **推理阶段**：仅需提供目标形状和提示文本（如“Evening Peak in Energy”），UPLOTS即可利用已学知识生成符合约束的新序列，无需重新训练。
