## 摘要
大语言模型越来越多地被用于金融、医疗或环境监测等领域的自动化决策系统。时间序列数据在这些领域中无处不在，但难以自动处理。时间序列能否由大语言模型代理进行分析？我们考察了三种方法：向代理提供原始数值数据、将大语言模型用作编码代理、以及两者的结合。在编码代理设置中，模型使用Python代码迭代地查询数据。通过两个时间序列理解基准测试，我们发现具有代码访问权限的代理在性能上可以超越处理原始数据的模型高达10%。然而，即使是最佳表现的代理，仍然有大约22-34%的问题回答错误。为了深入了解模型的策略和推理差距，我们使用一个强大的大语言模型评判器来分析模型输出。我们的分析表明，编码代理能够选择适当的统计测试，但常常忽略重要的细微差别。同时，具有原始数据访问权限的模型可以使用粗略估算得出正确的结论。

## Abstract
Large language models (LLMs) are increasingly being used for automated decision-making systems in finance, healthcare, or environmental monitoring. Time series data are ubiquitous in these fields, yet hard to process automatically. Can time series be analyzed by LLM agents? We examine three approaches: providing the agent with raw numerical data, using the LLM as a coding agent, or a combination of both. In the coding agent setup, the model iteratively queries the data using Python code. Using two time series understanding benchmarks, we show that agents with code access can outperform models processing raw data by up to 10%. However, even the best performing agent still answers about 22-34% of the questions incorrectly. To get insights into models' strategies and reasoning gaps, we analyze the model outputs with a strong LLM judge. Our analysis reveals that coding agents can select appropriate statistical tests, but often miss important nuances. Meanwhile, models with access to raw data can reach the right conclusions using back-of-the-envelope calculations.

### 6. 主要结论与发现

1. **编码代理整体更优**：Mcode 和 Mhybrid 的准确率比 Mdirect 高出最多 10%（例如 TSE 上 gpt-oss-120b：Mhybrid 78.0% vs Mdirect 65.3%）。
2. **混合设置（Mhybrid）最稳定**：在两种数据集上均最佳，同时利用原始数据的直觉和代码工具的精确计算。
3. **编码代理并非总可靠**：即使代码执行成功，仍有 25%（Mcode）/19%（Mhybrid）的答案存在“策略内方法选择错误”，常见错误为设置任意阈值、忽略相位移、忽略高次谐波等。
4. **简单算术方法使用广泛**：占所有答案的 47.4%，即使在编码代理中也占 27.9%-31.6%。对于噪声理解、异常检测、模式识别等任务效果较好。
5. **直接模型（Mdirect）缺陷在于策略选择错误**：33% 的答案选择了不合适的核心策略（如需要统计检验时仅用目测），导致准确率低下。
6. **编码代理过度依赖代码输出**：当代码失败或未使用代码时，准确率骤降至接近随机（Mcode 无代码时 42.0%）。
7. **失败模式多样**：包括概念误解、不支持的假设、结果误读、推理-答案不匹配、幻觉数值等（见图 4）。
8. **资源消耗差异**：Mdirect 消耗更多 token（TSE 上平均 13,340 vs Mcode 4,036），但准确率更低；而 Mhybrid token 效率与 Mcode 相当。

### 7. 优点

- **系统性的三种设置对比**：清楚地分离了代码能力和原始数据各自的作用，消融设计干净。
- **深层次的行为分析**：不只是报告准确率，而是通过分类法和 LLM 评判器深入挖掘模型的推理策略和失败原因，提供了模型“如何思考”的视角。
- **评判器验证严谨**：通过人工标注和 κ 系数选择了与人类判断最一致的评判器，并检查了自偏好偏差，增强了分析的可信度。
- **开源复现性**：提供了实验代码和模型输出，便于后续研究复现和扩展。
- **覆盖多类问题**：在两个数据集上按子类报告结果，揭示了问题类型对模型性能的显著影响。

### 8. 不足与局限

- **基准覆盖范围有限**：仅使用多项选择问答，不能反映真实环境中开放式的分析任务（如编写完整分析报告、处理缺失值、多表关联等）。作者承认这是“理解性能的下限”。
- **模型时效性**：只测试了两个开源模型（2025-2026 年发布），更新更快的模型可能已经解决了部分问题。
- **未涉及商业模型**：由于需要完全访问推理轨迹，无法使用 Claude/GPT-4 等商业模型（其推理链常被隐藏或受限），限制了结论的普适度。
- **未与专门时间序列模型对比**：如 TimesNet、Chronos 等模型可能在原始数据上表现更好，但本实验聚焦于通用 LLM。
- **评判器验证集规模小**：仅 36 例，可能不足以捕捉所有类型的行为模式；分类法本身也可能存在遗漏。
- **代码执行环境限制**：仅预装了少数库，可能限制了模型调用更合适算法的能力。
- **可能遗漏了视觉表示**：未考虑多模态设置（如提供时间序列图表），而视觉理解可能帮助模型避免部分错误。
