## 摘要
时间序列基础模型在给定非空历史窗口时表现出强大的迁移性能。然而，真正冷启动场景（新项目没有先验观测）违背了这一假设。我们提出RAID（检索增强迭代扩散）框架，用元数据驱动的语义检索和图条件扩散替代基于历史的相关性学习。RAID使用冻结的多语言嵌入模型将文本元数据映射到共享语义空间，并构建一个可自然地扩展到未见项目的归纳检索图。它首先通过聚合语义相关邻居的信息形成基础预测，然后使用门控扩散模块细化该预测以建模残差不确定性。在严格真正冷启动协议下，RAID在预测精度和预测区间覆盖率上均优于强基础模型和竞争基线，并通过非自回归解码将推理延迟降低一个数量级。共享语义空间还实现了零样本跨语言迁移，使得在英文描述上训练的模型能够推广到用其他语言描述的项目，而无需直接监督。

## Abstract
Time-series foundation models show strong transfer performance when given a non-empty history window. However, true cold-start scenarios, where a new item has no prior observations, violate this assumption. We propose RAID (Retrieval-Augmented Iterative Diffusion) a framework, which replaces history-based correlation learning with metadata-driven semantic retrieval and graph-conditioned diffusion. RAID maps textual metadata into a shared semantic space using a frozen multilingual embedding model and constructs an inductive retrieval graph that extends naturally to unseen items. It first forms a base forecast by aggregating information from semantically related neighbors, then refines this forecast with a gated diffusion module to model residual uncertainty. Under a strict true cold-start protocol, RAID outperforms strong foundation models and competitive baselines on both forecasting accuracy and prediction interval coverage, while reducing inference latency by an order of magnitude through non-autoregressive decoding. The shared semantic space also enables zero-shot cross-lingual transfer, allowing a model trained on English descriptions to generalize to items described in other languages without direct supervision.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：现有时间序列基础模型（如 Chronos、TimesFM、Moirai）依赖非空历史窗口（`L > 0`）进行预测，在**真正冷启动（True Cold-Start）场景**下（即新项目没有历史观测数据，`L = 0`）会崩溃为全局均值或随机噪声。同时，这些模型的自回归架构带来二次计算复杂度，推理延迟高。
- **研究动机**：在现实应用中（如新品上市、电商产品上架），元数据（产品描述、品牌、类别）往往在第一个观测点之前就已存在，但现有模型无法利用这些信息替代缺失的历史数据。因此，需要一种**不依赖历史窗口、仅凭元数据即可进行高质量预测**的方法。
- **整体含义**：RAID 提出一种**解耦“记忆”（检索）与“生成”（扩散）** 的新范式，通过语义图驱动的检索与扩散模型，实现真正冷启动下的零样本预测，同时具备跨语言迁移能力和高效推理。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想
- **语义同质性（Semantic Homophily）**：元数据相似的物品往往具有相似的需求动态（如“耐克跑鞋”与“阿迪达斯跑鞋”季节性相似）。
- **解耦为两步**：先通过检索构建确定性基础预测，再通过扩散建模残差不确定性。

### 2.2 关键技术细节
1. **语义嵌入空间**：
   - 使用冻结的多语言嵌入模型 BGE-M3 将文本元数据（含类别、品牌、标题、描述字段）映射为 1024 维向量 `e_i`，并进行 L2 归一化。
   - 通过 Tanh 激活的瓶颈层降维至 16 维（`e'_i`）供下游模块使用。
2. **归纳图拓扑**：
   - 基于嵌入余弦相似度，为每个节点选取 Top-k 近邻，构建**稀疏图**。邻接权重采用 Top-k 稀疏 softmax 核（可学习温度 τ）。
   - 图是**归纳式**的：新节点只需一次前向传播即可确定邻居，无需重新训练。
3. **检索增强的基础预测**：
   - **形状-尺度分解**：为避免不同规模邻居的原始轨迹直接平均导致的尺度偏差（例如 Echo Dot 与 Kids Edition 销量差 112 倍），将邻居轨迹归一化为尺度不变的形状（z-score），并用 attention 权重聚合得到归一化先验 `z_agg`。
   - 通过 MLP 从瓶颈嵌入预测目标特定的尺度 `σ̂_i` 和均值 `μ̂_i`，重建基础预测：`μ_base = Softplus(σ̂_i) · z_agg + μ̂_i`。
4. **门控扩散 Transformer（DiT）**：
   - 残差目标 `r = y - μ_base` 由 DDPM 建模，采用 DiT 骨架（4 层 Transformer，自注意力）。
   - 条件注入通过**自适应层归一化（AdaLN）** 实现，图上下文 `c` 和时间步嵌入 `t_s` 共同调节每个隐藏层的归一化参数。
   - 门控融合：`ŷ = μ_base + sigmoid(MLP_ψ(c)) · r̂_0`，初始偏置设为 -3.0（sigmoid ≈ 0.047），使模型初期依赖基础预测，逐步学习门控。
5. **非自回归高效推理**：
   - 基础预测可预计算并哈希索引，实现 O(1) 检索。
   - 残差扩散仅需 S=10 步（得益于残差化降低熵），而原始数据扩散需 50-100 步。

### 2.3 算法流程（文字描述）
- **训练**：对每个 batch，随机掩码历史窗口模拟冷启动；检索邻居并计算形状-尺度基础预测；用 DiT 对残差进行去噪；联合优化扩散损失和基础预测 L2 损失。
- **推理**（冷启动 `L=0`）：嵌入查询元数据 → 检索邻居 → 预测尺度参数 → 聚合形状 → 生成基础预测 → 图编码上下文 → 扩散去噪 S=10 步 → 门控融合得到最终预测。

## 3. 实验设计

## 6. 主要结论与发现
1. **严格冷启动下领先**：RAID 在 9 个数据集上平均排名 1.3，sMAPE 在 6/9 个数据集上最优，CRPS 在 4/9 最优。基础模型（Chronos 排名 7.8、TimesFM 7.6）在 `L=0` 时崩溃至无条件先验。
2. **跨语言零样本迁移**：在英文训练→西班牙语、西里尔语、汉字等场景下，RAID 性能与原生训练差异在 ±3.6% 以内，无跨脚本崩溃。
3. **效率极高**：全概率版 RAID-Full 比 Chronos-Small 快 48.7 倍，确定性版 RAID-Base 快 124 倍；对 100 万物品，RAID-Base 约 2.9 小时，Chronos 需 360+ 小时。
4. **LLM 直接提示失败**：8 个大型语言模型在冷启动下普遍出现**规模幻觉**（预测量级偏差数十至上百倍），sMAPE 1.6-1.9，即使注入邻居销售数据也只能修正尺度但无法恢复形状（sMAPE 仍 1.18 对 RAID 的 0.51）。

## 7. 优点
- **创新范式**：解耦检索与扩散，巧妙利用语义同质性，解决了基础模型无法处理的真正冷启动问题。
- **架构精巧**：形状-尺度分解解决了邻居规模差异；门控融合平滑整合基础预测与扩散残差；AdaLN 条件注入高效且在训练初期稳定。
- **跨语言通用**：利用 BGE-M3 多语言嵌入，实现零样本跨语言迁移，无需目标语言数据。
- **工程友好**：推理延迟降低一个数量级，适合大规模高频预测场景；大部分组件可预计算。
- **理论根基**：用残差化降低扩散目标熵（Remark 3.1），为小步数推理提供了理论动机。
- **实验严谨**：统计显著性检验、Bootstrap 置信区间、多种子报告，超参数固定避免过拟合。

## 8. 不足与局限
1. **三大前提假设**：
   - **语义同质性**：若元数据相似但动态无关（如“苹果”水果 vs “苹果”公司）会失效，虽由 GAT 减弱但无法完全消除。
   - **静态语义**：元数据与动态关系需稳定；COVID-19 等外部冲击会破坏 Lipschitz 假设。
   - **高斯残差**：对重尾或多模态残差（间歇性需求）可能需要更多扩散步数，当前 S=10 可能不足。
2. **通用协议下非最优**：当完整历史可用时，RAID 平均排名 3.9，落后于 LightGBM（2.1）和部分基础模型（3.8）。RAID 专为冷启动设计，非通用场景。
3. **图神经网络过平滑风险**：在稀疏元数据数据集（Job-SDF）上去除 GATv2 反而提升 1.5% sMAPE，表明 GATv2 在弱同质性时可能过正则化，当前无自适应门控。
4. **跨语言迁移机制分析较浅**：仅提出“源图密度”假设，未做受控密度消融实验验证。
5. **社会影响**：冷启动预测能力可能被用于价格歧视，需配套审计机制。论文已正视此风险但未提供具体缓解方案。
6. **基础模型零样本评估公平性**：因架构缺乏元数据通道，仅零样本评估（而非微调）虽合理但可能未充分展示基础模型潜力。论文在附录 D.3 中进行了详细说明，自洽。
