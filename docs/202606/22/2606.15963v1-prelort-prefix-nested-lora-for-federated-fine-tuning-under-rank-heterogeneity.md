## 摘要
使用LoRA等参数高效方法对大型语言模型进行联邦微调，可以保护基础模型隐私的同时进行适配。异构硬件资源带来了挑战，因为不同适配器秩的客户端无法直接聚合。现有方法虽然实现了异构秩下的聚合，但未能控制信息在秩维度上的分布，导致共享低秩表示的利用不够优化。为此，我们提出PreLort：一种用于联邦LoRA的嵌套低秩公式，将适配器维度组织成前缀层次结构。我们的方法确保较低秩维度编码任务相关信息，而较高秩维度捕获额外容量。在此基础上，我们引入了(i)分段聚合规则，仅对每个秩段贡献的客户端进行平均，避免来自零填充低秩客户端的稀释，以及(ii)前缀嵌套训练策略，在多个秩截断下优化每个适配器，鼓励有用信号集中在低秩前缀维度中。这些组件共同促进一致的低秩前缀捕获最多任务相关信息，而较高秩维度学习额外容量。这使得低秩客户端能够从高秩客户端贡献的更丰富信息中受益，因为前缀维度被一致地学习和聚合。实验表明，我们的方法在准确性和ROUGE-L上始终优于先前的异构联邦LoRA方法，并且在多个基础模型上实现了更低或相当的困惑度。

## Abstract
Federated fine-tuning of large language models using parameter-efficient methods such as LoRA enables privacy-preserving adaptation of foundation models. Heterogeneous hardware resources introduce challenges, as clients with different adapter ranks cannot be directly aggregated. While existing methods enable aggregation under heterogeneous ranks, they fail to control how information is distributed across rank dimensions, leading to suboptimal use of shared low-rank representations. Instead, we propose PreLort: a nested low-rank formulation for federated LoRA that organizes adapter dimensions into a prefix hierarchy. Our approach ensures that lower-rank dimensions encode task-relevant information, while higher-rank dimensions capture additional capacity. Building on this, we introduce (i) a segment-wise aggregation rule that averages only over clients contributing to each rank segment, avoiding dilution from zero-padded lower-rank clients, and (ii) a prefix-nested training strategy that optimizes each adapter under multiple rank truncations, encouraging useful signal to concentrate in low-rank prefix dimensions. Together, these components encourage a consistent low-rank prefix capturing the most task-relevant information, while higher-rank dimensions learn additional capacity. This allows low-rank clients to benefit from richer information contributed by higher-rank clients, as prefix dimensions are consistently learned and aggregated. Experiments demonstrate that our method consistently outperforms prior heterogeneous federated LoRA methods in accuracy and ROUGE-L, while achieving lower or comparable perplexity across multiple base models.

# PreLort：面向秩异构联邦微调的前缀嵌套LoRA —— 论文深度总结

## 1. 核心问题与研究动机

- **背景**：联邦学习（FL）中，客户端使用LoRA等参数高效微调（PEFT）方法对大语言模型进行微调，可保护隐私并降低通信/计算成本。然而实际场景中，客户端硬件资源（内存、算力）不同，导致各客户端所能使用的LoRA秩（rank）不同，即存在**秩异构**问题。
- **现有方法的局限**：当前方法（如ZeroPad、HetLoRA、FLoRA）虽能通过零填充或堆叠等方式实现异构秩的聚合，但**未控制信息在秩维度的分布**，导致低秩客户端无法有效利用高秩客户端学到的丰富信息，且聚合时因零填充而稀释高秩更新的贡献。
- **关键洞察**：需要同时解决**表示对齐**（不同秩维度间信息结构对齐）、**训练动态**（使低秩前缀编码核心任务信息）和**聚合策略**（避免零填充稀释）三个问题，而非仅关注聚合操作。

## 2. 方法论：核心思想与技术细节

### 2.1 核心思想
- **前缀嵌套低秩结构**：将LoRA适配器的维度组织成前缀层次——**低秩维度编码任务相关信息**，高秩维度提供额外容量。训练时强制每个适配器在多个前缀截断（sub-rank）下都保持有效，从而让重要信息集中到前缀的低秩部分。
- **两个关键组件**：
  - **分段聚合（Segment-wise Aggregation）**：每个秩段只对支持该段的客户端（即秩足够的客户端）进行平均，避免零填充污染。
  - **嵌套训练（Nested Training）**：每个客户端在每个批次中同时计算全秩和多个前缀子秩下的损失，并取平均，迫使低秩前缀收敛到共享的有用表示。

### 2.2 技术细节
- **嵌套低秩结构**：对于秩为 \(r_k\) 的客户端，定义活跃训练秩集 \(R_k = \{ r \in R \mid r \le r_k \}\)，其中 \(R\) 是系统中所有不同秩的集合。训练时对每个 \(r \in R_k\)，将适配器截断为前 \(r\) 维的 \(A^{(r)}_k, B^{(r)}_k\)，计算交叉熵损失，然后平均所有子秩损失作为总损失。
- **分段聚合**：令 \(r_0=0, r_1 < r_2 < ... < r_L\) 为所有客户端的不同秩。对于第 \(l\) 段 \([r_{l-1}, r_l)\)，仅在秩≥\(r_l\) 的客户端上聚合伪梯度（参数更新量）：
  \[
  \nabla A_{\text{global}}^{[r_{l-1}:r_l]} = \frac{\sum_{k: r_k \ge r_l} n_k \nabla A_k^{[r_{l-1}:r_l]}}{\sum_{k: r_k \ge r_l} n_k}
  \]
  B矩阵类似。这保证了每个段只由有能力的客户端贡献。
- **训练流程（Algorithm 1 & 2）**：每个客户端本地执行多轮嵌套训练，生成伪梯度（初始与训练后参数差）；服务器按段聚合伪梯度并更新全局适配器。

## 3. 实验设计

### 3.1 数据集与基准
- **数据集**：
  - 指令微调：**Alpaca**（52K 样本）、**Databricks-dolly-15**（15K 样本）
  - 分类：**20 Newsgroups**（约20K文档，20类）
- **评估指标**：
  - **Accuracy**：MMLU（Dolly、Alpaca 子集）和 20 Newsgroups 分类准确率
  - **ROUGE-L**：指令微调数据集上的文本生成质量
  - **Perplexity**：语言模型 token 级似然
- **基础模型**：TinyLlama-1.1B、Qwen2.5-0.5B

### 3.2 对比方法
- **ZeroPad**：FedAvg + 零填充对齐异构秩
- **HetLoRA**：秩自剪枝 + 稀疏加权聚合
- **FLoRA**：基于堆叠（stacking）的精确更新聚合，避免交叉项
- **PreLort**（本文方法）
- **消融变体**：
  - Nested Aggregation Only（仅分段聚合，无嵌套训练）
  - Nested Training Only（仅嵌套训练，但聚合使用零填充 FedAvg）

### 3.3 实验设置
- 异构配置：10 客户端场景（秩分布 [64,32,16,16,8,8,4,4,4,4]）和 5 客户端场景（[16,8,4,2,1]），遵循 Wang et al. (FLoRA) 设置。
- 通信轮次：15 轮；客户端每轮本地训练 1~3 个 epoch（取决于数据集）。
- 优化器：AdamW，学习率 3e-4，有效 batch size 64。
- LoRA 仅应用于自注意力层。
