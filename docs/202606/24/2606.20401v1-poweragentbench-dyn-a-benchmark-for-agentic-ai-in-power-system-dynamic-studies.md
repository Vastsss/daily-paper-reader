## 摘要
基于大语言模型的智能体正越来越多地通过交互软件工具、解读中间结果并自主规划后续行动，来自动化多步骤工程工作流。电力系统动态研究是这些智能体特别有前景但尚未充分探索的应用领域。与静态计算任务不同，动态研究通常需要更多时间进行模型参数校准、工程判断以及在受限动作空间下的决策。本文介绍PowerAgentBench-Dyn，这是一个旨在评估电力系统动态分析任务中智能体AI系统的基准测试。该基准测试针对那些不能简化为单一优化或编码任务的问题，而是需要经验丰富的电力系统工程师通常执行的一种推理、工具使用和迭代实验。所提出的框架包括两个初始基准任务。第一个是动态模型质量审查基准，评估智能体基于系统运营商规定的模型质量合规标准来验证和诊断动态模型的能力。第二个是动态安全风险筛查基准，评估智能体利用语义记忆和有限的仿真预算从未知故障数据集中识别、排序和分析最关键短路事故，以及提出并评估可能的缓解措施的能力。对于每个任务，我们定义了仿真环境、观测和动作空间以及评估指标。该基准测试在基于指标的意义上是可复现的：发布的案例和仿真器设置定义了一个确定性评估器，而随机智能体行为则通过多次运行的成功率和其他指标进行评估。该基准测试支持未来用于电力系统运行和规划的智能体AI的开发。

## Abstract
Large Language Model (LLM)-based agents are increasingly being used to automate multi-step engineering work flows by interacting with software tools, interpreting intermediate results, and autonomously planning subsequent actions. Power system dynamic studies represent a particularly promising yet largely unexplored application domain for these agents. Unlike static computational tasks, dynamic studies often require more time on model parameter calibration, engineering judgment, and decision making under constrained action spaces. This paper introduces PowerAgentBench-Dyn, a benchmark designed to evaluate Agentic AI systems on power system dynamic-analysis tasks. The benchmark targets problems that cannot be reduced to a single optimization or coding task, but instead require a type of reasoning, tool usage, and iterative experimentation routinely performed by experienced power system engineers. The proposed framework includes two initial benchmark tasks. The first, the Dynamic Model Quality Review Benchmark, evaluates agents' ability to validate and diagnose dynamic models based on model-quality compliance criteria specified by system operators. The second, the Dynamic Security Risk Screening Benchmark, assesses agents' capability to leverage semantic memory and a limited simulation budget to identify, rank, and analyze the most critical short-circuit contingencies from an unseen fault dataset, as well as propose and evaluate possible mitigation measures. For each task, we define the simulation environment, observation and action spaces, and evaluation metrics. The benchmark is reproducible in a metric-based sense: released cases and simulator settings define a deterministic evaluator, while stochastic agent behavior is assessed over repeated runs using success rates and other metrics. The benchmark supports the development of future Agentic AI for power system operation and planning.

## 2. 论文提出的方法论

### 2.1 核心思想
每个基准任务建模为一个交互式环境：
\[
T = (O, A, E, C, G, M)
\]
其中O为观测空间，A为受约束的动作空间，E为仿真环境，C为用户和工程约束集，G为隐藏的真实值或评估器，M为可选的语义记忆。

- 智能体在每一步选择动作，获得仿真器反馈，直到返回最终答案或达到动作/时间预算。
- 动作空间故意受限：智能体只能修改用户授权的参数，不能任意改动模型结构或锁定文件。

### 2.2 关键技术细节
- **工具接地**：智能体必须通过工具（如DMView、PSS/E、DIgSILENT PowerFactory）与仿真器交互。
- **多轮决策**：任务需要多轮交互，而非单次响应。
- **工程约束护栏**：动作必须在允许的动作空间、预算和锁定文件内。
- **面向结果的指标**：成功由可靠性相关结果（如诊断准确性、波形拟合、排名质量、缓解成功率）衡量。

### 2.3 两个基准任务
1. **动态模型质量评审（DMQ）**：智能体使用DMView工具运行模型质量测试，根据pass/fail报告诊断问题，然后仅修改用户授权的4个REECAU1控制器增益（Kqp, Kqi, Kvp, Kvi），迭代直至所有测试通过或达到最大迭代次数（5次）。
2. **动态安全风险筛查（DSR）**：智能体查询语义记忆，从10个候选短路事故中选择最关键的3个进行仿真，计算KPI（电压极值、稳定性标志、严重度分数），并评估可能的缓解措施（同步调相机安装），受限于固定仿真预算。

### 2.4 评估指标
- DMQ：全测试集通过率、波形拟合、LVRT恢复、耗时、约束合规。
- DSR：top-k重叠、严重度分数绝对误差、缓解成功率、耗时、约束合规。

## 3. 实验设计

### 3.1 数据集/场景
- **DMQ**：使用修改后的WECC太阳能光伏动态模型，增益设为激进值(10,50,10,50)，导致所有8项模型质量测试失败。输入包括PSS/E潮流案例、Solar.dyr文件、DMView测试配置、动作空间描述及5次迭代预算。
- **DSR**：使用IEEE标准测试系统（未具体说明，但提到IEEE 9-bus、39-bus等作为可扩展的测试平台）。10个候选三相短路事故，仿真离线确定真实值，三个最严重事故为L02–25@160ms（不稳定）、L05–06@200ms、B05@180ms。语义记忆包含类似案例的仿真结果。

### 3.2 基准对比的方法
- **DMQ**：对比Anthropic的三个模型：Opus 4.8、Sonnet 4.6、Haiku 4.5。
- **DSR**：对比六个模型：Anthropic的Opus 4.7、Sonnet 4.6、Haiku 4.5；Google的Gemini 3.1 Pro；OpenAI的GPT-5.5；阿里巴巴的Qwen2.5-Coder-7B。

### 3.3 实验设置
- 每个模型运行10次独立会话（基于LLM采样的随机性）。
- 使用自动化测试框架：DMQ通过工具暴露DMView，DSR通过MCP框架连接DIgSILENT PowerFactory。

## 6. 论文的主要结论与发现

- **DMQ任务**：Opus 4.8和Sonnet 4.6在10/10次中成功修复模型，通常一次更新即可；Haiku 4.5成功9/10次，但需中位数2次更新且有一次失败。这表明基准能区分模型在受限动作空间下定位问题参数的效率和可靠性。
- **DSR任务**：Opus 4.7、Sonnet 4.6、Gemini 3.1 Pro、GPT-5.5完成工作流成功率≥90%，且准确识别三个最严重事故，正确标记不稳定事故L02–25@160ms，电压极值估计误差小。Haiku 4.5仅70%成功，主要卡在工具执行而非推理。Qwen7B完全失败（无法调用工具）。
- **缓解分析**：所有成功模型一致发现仅安装SC04无效，必须包含SC26才能稳定L02–25@160ms事故。
- **运行时**：成功模型在5-12分钟内完成DSR任务。

## 7. 优点

- **任务设计贴切实用**：选择电力系统工程师日常面临但难以自动化的开放性问题（模型参数校准、有限预算下事故排序），而非简单分类或回归。
- **约束导向**：明确限制动作空间（如仅允许修改4个增益），模拟真实工业场景（不能随意改动模型），避免智能体走捷径。
- **可重复性设计**：确定性地面真值评估器 + 多次随机运行统计指标，使结果稳定可比。
- **工具集成自然**：DMQ使用真实工业工具DMView，DSR使用MCP框架连接业界通用仿真软件DIgSILENT PowerFactory，增加实用价值。
- **指标多元**：不仅看最终成功率，还看迭代次数、波形拟合、约束合规、过程质量，防止智能体通过违规修改达到伪成功。

## 8. 不足与局限

- **实验覆盖有限**：仅测试了少数LLM模型（主要是闭源），且未包括更多开源模型（如Llama 3、DeepSeek等）。动作预算（5次迭代、仿真次数）固定，未探讨不同预算下的表现。
- **简化的真实世界**：任务假设模型文件可访问、没有保护电路细节、没有EMT/RMS模型差异、没有数据保密问题。实际工业场景中这些因素会显著增加复杂度。
- **单层智能体设计**：论文仅测试了基础LLM工具调用，未探索更鲁棒的智能体架构（如重试机制、验证器、多智能体协作）。作者承认这一点，并留有改进空间。
