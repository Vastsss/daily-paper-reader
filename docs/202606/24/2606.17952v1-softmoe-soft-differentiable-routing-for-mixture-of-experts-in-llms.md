## 摘要
稀疏专家混合（MoE）架构通过仅激活每个输入的一小部分专家（通过 top-k 路由），在固定推理预算下扩展大语言模型参数。虽然这保持了因果性并适用于自回归语言模型，但离散的 top-k 算子不可微，迫使每个输入使用固定数量的活跃专家，导致计算效率低下。我们提出 SoftMoE，它用截断的软 top-k LapSum 松弛替换离散路由，允许基于梯度的专家路由优化。我们进一步参数化每层的平均活跃专家数量，并施加全局预算约束，使模型能够学习如何跨层分配专家容量。SoftMoE 与自回归建模完全兼容，在语言建模和下游任务上实现了与稀疏 MoE 相当或更好的性能，同时激活的专家显著减少。值得注意的是，学习到的分配高度不均匀，后面层激活更多专家。源代码已公开$^\dagger$。

## Abstract
Sparse Mixture-of-Experts (MoE) architectures enable scaling LLM parameters under a fixed inference budget by activating only a small subset of experts via top-$k$ routing. While this preserves causality and suits autoregressive language models, the discrete top-$k$ operator is not differentiable, forcing a fixed number of active experts per input and resulting in inefficient use of computation. We propose SoftMoE, which replaces discrete routing with a truncated soft top-$k$ LapSum relaxation, allowing gradient-based optimization of expert routing. We further parameterize the mean number of active experts per layer and impose a global budget constraint, enabling the model to learn how to allocate expert capacity across layers. SoftMoE remains fully compatible with autoregressive modeling and achieves performance comparable to or better than sparse MoE on language modeling and downstream tasks, while activating significantly fewer experts. Notably, the learned allocation is highly non-uniform, with later layers activating more experts. The source code is publicly available$^\dagger$.

# SoftMoE: 用于大语言模型专家混合的软可微路由

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：传统稀疏专家混合（MoE）架构使用离散的 top-k 路由，即每个 token 仅激活得分最高的 k 个专家。该操作不可微，导致梯度无法通过路由选择进行优化；同时专家激活数固定，无法根据输入动态调整，造成计算效率低下。
- **研究动机**：希望设计一种可微分、自适应、不破坏自回归因果性的路由机制，使模型能够按 token 和按层动态分配专家容量，在保持或提升性能的同时降低计算成本。
- **整体含义**：提出 SoftMoE，用软 top-k 松弛替代硬选择，首次实现 MoE 中专家分配的端到端梯度学习，并引入全局预算约束下的层间专家分配学习，显著提升计算效率。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：用 LapSum 函数构造可微分的软 top-k 算子（SoftTopK），生成连续软路由权重（求和为 k，元素在 [0,1]），并通过截断（truncation）恢复稀疏性；进一步将每层平均活跃专家数 k_l 参数化，施加全局预算约束，使模型自动学习跨层分配。
- **关键技术细节**：
  - **SoftTopK 算子**：基于 LapSum 函数（Laplace CDF 之和）求解连续阈值 $\tilde{x}$，使得 $\sum_i F_{\text{Lap}}(r_i - \tilde{x}) = k$，软权重 $\tilde{p}_i = F_{\text{Lap}}(r_i - \tilde{x})$。该算子具有 O(n) 复杂度，完全可微。
  - **截断（Truncation）**：对软权重施加阈值 $\tau$，仅保留大于 $\tau$ 的权重，恢复稀疏性并使激活专家数输入依赖。截断处梯度为零，但模型仍可通过 k_l 的梯度学习分配。
  - **层间专家分配学习**：用参数 $\eta_l$ 经 softmax 重参数化得到 $k_l = \pi_l \cdot (K - L) + 1$，其中 $\sum k_l = K$ 为全局预算，$k_l \ge 1$。该重参数化完全可微，可联合优化。
  - **辅助损失**：使用 Switch Transformer 的负载均衡辅助损失；预激活加噪声以鼓励探索。
- **算法流程**（文字描述）：
  1. 每个 token 通过门控网络得到路由分数 $r = xW_g$；
  2. 使用 SoftTopK（LapSum）将 r 映射为软权重 $\tilde{p}$（求和为 $k_l$）；
  3. 应用截断 $T(z) = z \cdot I_{z > \tau}$，得到稀疏路由权重；
  4. 按权重加权组合专家输出：$y = \sum_i T(\tilde{p}_i) \cdot E_i(x)$；
  5. 训练时通过重参数化参数 $\eta$ 学习每层 $k_l$，受全局预算 $K$ 约束。

## 6. 主要结论与发现

- **效率提升**：SoftMoE* 在激活更少专家的同时，语言建模损失低于或等于稀疏 MoE。例如，≤2 预算下激活专家数减少 17%~24%，损失更低。
- **学习到的分配模式**：SoftMoE 自动将专家预算向后期层集中（顶层 3 层约吸收 50% 预算），且该模式在训练早期（25k 步内）迅速形成。后期层受益于更多专家容量，符合深层编码语义特征的认知。
- **下游性能**：在 HellaSwag 和 PIQA 上，SoftMoE 普遍优于或持平稀疏 MoE；ARC-E 上虽方差较大，但 SoftMoE 仍具竞争力。
- **可微路由优势**：软路由使专家激活数随输入变化，简单 token 激活少量专家，复杂 token 利用更多专家，避免固定预算浪费。

## 7. 优点

- **方法创新**：首次将可微 top-k 松弛（LapSum）成功应用于 MoE 路由，保持稀疏性和因果性，解决长期存在的不可微问题。
- **学习分配机制**：通过全局预算约束实现层间专家容量自动分配，发现非均匀分配规律，为设计深度感知 MoE 提供依据。
- **实验严谨**：涵盖两个不同规模数据集、多个预算配置、消融分析和下游评测，对比公平。
- **效率显著**：在性能持平或更优前提下，激活专家数减少达 17%~24%，推理成本降低。
- **代码开源**：便于复现和应用。

## 8. 不足与局限

- **语料局限性**：仅评估英语文本（OWT 和 C4），未涉及多语言场景。
- **任务范围有限**：未测试多模态、机器翻译等更广泛任务。
- **模型规模**：1.63B 参数小于实际部署的大型模型（如数百 B），扩展性需进一步验证。
- **截断阈值选择**：τ 需通过统计初始批次确定，且影响性能（附录表 6），存在调参开销。
- **理论分析欠缺**：未给出 LapSum 松弛的逼近误差界或收敛性保证。
- **计算开销**：虽然专家激活减少，但 SoftTopK 的 O(n) 操作引入额外内存和计算（相对稀疏 MoE 的 O(k)），在极大规模 n 时可能成为瓶颈。
