## 摘要
时间序列通常嵌入在丰富的上下文中，这对于整体建模至关重要。此外，实际从业者通常需要端到端的工作流来分析时间动态，其中广泛研究的任务（如预测）只是更广泛解决方案循环中的一个步骤。虽然通用AI代理为此类工作流提供了有前景的接口，但它们主要在文本空间中操作，并未完全与结构化时间信号对齐。在这项工作中，我们引入了TimeClaw，一个用于时间序列的代理框架，该框架为通用大语言模型代理提供了上下文时间推理所需的时间序列原生运行时支持。TimeClaw集成了可执行的时间工具，用于基于事实和可审计的分析；基于经验的能力演化，用于创建可重用的分析例程；以及情景多模态记忆，用于检索相关的推理轨迹。这些组件共同实现了利用上下文信息的开放式时间推理。在涵盖能源、金融、天气、交通及其他实际领域多种任务的多个基准上的广泛评估表明，TimeClaw的性能有所提升。代码可在https://github.com/iDEA-iSAIL-Lab-UIUC/TimeClaw获取。

## Abstract
Time series are often embedded in rich contexts that are essential for holistic modeling. Moreover, real-world practitioners often require end-to-end workflows for analyzing temporal dynamics, where widely studied tasks such as forecasting are only one step in a broader solution loop. While generalist AI agents offer a promising interface for such workflows under complex contexts, they still operate primarily in textual spaces that are not fully aligned with structured temporal signals. In this work, we introduce TimeClaw, an agentic harness framework for time series that equips generalist LLM agents with the time series-native runtime support needed for contextualized temporal reasoning. TimeClaw integrates executable temporal tools for grounded and auditable analysis, experience-driven capability evolution for creating reusable analytical routines, and episodic multimodal memory for retrieving relevant reasoning traces. Together, these components unlock harnessed open-ended temporal reasoning with contextual information. Extensive evaluation on multiple benchmarks covering diverse tasks across energy, finance, weather, traffic, and other real-world domains demonstrates improved performance of TimeClaw. Code is available at https://github.com/iDEA-iSAIL-Lab-UIUC/TimeClaw.

# 论文总结：Harnessing Generalist Agents for Contextualized Time Series（TimeClaw）

## 1. 核心问题与研究动机

- **问题背景**：实际时间序列分析常需结合丰富的上下文信息（如文本描述、领域知识、外部事件），并需要端到端工作流（不仅仅是预测）。通用大语言模型（LLM）代理虽擅长文本推理，但直接处理数值时间序列存在两种根本错配：
  - **数据类型错配**：数值序列经序列化后以离散文本令牌表示，会损失数值连续性、精度和时序结构（如趋势、周期）。
  - **代理过程错配**：现有代理框架以文本为中心，无法原生执行时间序列操作（切片、聚合、分解等），导致长上下文稀释和推理不可靠。
- **研究目标**：构建一个**时间序列原生的代理框架（harness）**，使通用LLM代理能可靠、可审计地推理上下文时间序列。

## 2. 方法论：TimeClaw 框架

TimeClaw 为冻结的LLM策略提供了一个时间序列原生运行时环境，包含三个协调组件：

### 核心组件
- **可执行时间工具（Executable Temporal Tools）**：
  - 所有工具直接在运行时工作空间（Workspace）内以全数值精度执行，返回结构化观测（而非文本）。
  - 初始工具包括：通道列表、概览统计、自相关、频谱、峰值检测、ARIMA预测等（见表10）。
  - 每个数值主张必须由工具观测支持，形成可审计的分析轨迹。
- **经验驱动能力演化（Experience-Driven Capability Evolution）**：
  - 通过演化算子E（由代码专用LLM实现）从多轮成功轨迹中识别重复子过程，并自动封装成可调用的工具（如金融领域的`portfolio_sharpe`、`portfolio_var`、`capm_regression`）。
  - 新工具需在预留任务上通过成功率阈值γ=0.7的验证后才被加入工具集。
- **情景多模态记忆（Episodic Multimodal Memory）**：
  - 记忆记录包含任务、轨迹、以及**文本键（上下文嵌入）**和**时间序列键（指纹）**。
  - 时间序列指纹（20维）融合结构、统计、频谱、动态和多元特征（见表4）。
  - 检索时采用**两阶段策略**：先基于文本余弦相似度筛选候选池（N=20），再基于归一化指纹L2距离重排序，返回top-k个记录。

### 工作流程
给定任务τ=(q, X, Y, y*, ℓ)，数值序列加载到工作空间；LLM通过工具与运行时交互生成轨迹ζ = (a1, o1, …, aK, oK, ŷ)，最终输出答案。所有操作均记录，支持审计和复用。

## 3. 实验设计

### 数据集与基准
- **Context-is-Key (CiK)**：71个任务，覆盖7大领域（公共安全、气候、交通、能源、零售、机械、经济），包含5种上下文类型（历史、未来、协变量、因果、无条件）。
- **TSRBench**：4125个实例，15个子任务，涵盖感知、推理、预测、决策四类能力，跨14个领域。因预算采用20%分层子集（825实例）。
- **TSAIA Finance**：150个金融时间序列分析实例（VaR、夏普比率、Alpha/Beta计算）。

### 对比方法
- **传统时间序列模型**：ARIMA, ETS, DLinear, PatchTST
- **时间序列基础模型**：Chronos系列, Lag-Llama, Moirai系列
- **时间序列LLM/代理**：UniTime, Time-LLM, TS-Agent, TSci
- **通用代理流水线**：直接提示, CoT, ReAct, 自反思, 多代理反思
- **开源模型**：Qwen2.5/3系列, Gemma3, InternVL3.5, Phi4-Multimodal, TS-Reasoner, TimeOmni等（用于TSRBench对比）
- **金融专用代理**：FinCon, Fin-Team, TradingAgents（用于TSAIA对比）

所有代理方法在CiK和TSRBench上使用相同基础LLM（GPT-5-nano），确保公平。

## 6. 主要结论

- TimeClaw在所有三个基准上显著优于现有方法（CiK上RCRPS降低11.5%，TSRBench上平均准确率提高15.8%，TSAIA上准确率提高38.9%）。
- **上下文利用是关键**：通过案例证明，相同的数值序列仅因上下文不同，TimeClaw能产生截然不同的正确预测。
- **三个组件均有效**：工具、能力演化、记忆各自贡献性能，移除任一个均造成下降。
- **记忆提高了效率**：检索相关轨迹不仅提升准确率，还减少不必要的工具调用（如从22次降至7次）。
- **能力演化实现了领域专长大**：金融环境中自动演化出专业工具（如CAPM回归）并提升应用性能。

## 7. 优点

- **创新性**：首次提出时间序列原生的代理框架，有效桥接文本与数值模态。
- **可审计性**：每个数值主张都有工具调用记录，避免LLM幻觉，适合高风险应用。
- **灵活性**：工具集可通过演化自动扩展，无需人工设计所有任务特定工具。
- **实验全面**：在多个真实世界基准上评估，对比方法涵盖传统模型、基础模型、代理系统、开源模型。
- **开源实现**：代码公开，便于复现和扩展。
- **案例研究深入**：定量展示上下文和记忆如何改变决策过程，有说服力。

## 8. 不足与局限

- **基础LLM依赖**：实验主要使用商业API（GPT-5系列），未探索完全开源方案；成本与延迟可能限制部署。
- **记忆构建成本**：需要预先收集成功的训练轨迹并构建记忆银行，这在全新领域可能不可行。
- **演化验证有限**：能力演化仅针对金融领域验证，未在更多领域展示通用性；演化出工具的质量受限于轨迹数量和LLM提炼能力。
- **基准覆盖**：TSRBench仅用20%子集，可能遗漏长尾难度场景；CiK任务数量有限（71个），统计显著性可能受限。
- **未涉及长序列或高维场景**：实验中的时间序列长度和维度相对有限，未测试极长序列或数百通道的场景。
- **缺少公平性/鲁棒性分析**：未讨论上下文中的错误或误导信息对结果的影响，也未评估对数值噪声的鲁棒性。
- **遗忘风险**：记忆检索依赖文本和指纹相似性，可能存在误召回导致的负迁移（尽管通过两阶段策略缓解）。
