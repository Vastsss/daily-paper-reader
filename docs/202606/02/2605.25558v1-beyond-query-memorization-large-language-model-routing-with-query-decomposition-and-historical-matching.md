## 摘要
在部署大语言模型（LLMs）时，优化预测性能与计算成本之间的权衡是一个核心关注点。当前的路径选择方法主要依赖于基于表面特征从查询到模型的直接映射，这使得它们容易陷入记忆化陷阱，并导致在分布外（OOD）数据上泛化能力差。在本文中，我们提出DecoR，一种新颖的路径选择框架，它将路径选择任务重新定义为从历史日志中筛选相似查询的匹配过程，有效缓解了记忆化陷阱。为了提高匹配准确性，我们引入了一种查询能力解构方法，将语言表面形式与任务固有需求分离，将匹配导向能力维度，以基于关键任务属性进行决策。此外，我们开发了CodaSet，一个用于评估路径选择泛化能力的综合基准，实验结果表明，DecoR在分布内和分布外设置中均保持了优越的准确性，同时大幅降低了推理成本。所有代码和数据均可在https://github.com/lvbotenbest/DecoR获取。

## Abstract
Optimizing the trade-off among predictive performance and computational cost is a central focus in the deployment of Large Language Models (LLMs). Current routing methods primarily rely on direct mapping from queries to models based on surface-level features, making them susceptible to the memorization trap and leading to poor generalizability on out-of-distribution (OOD) data. In this paper, we propose DecoR, a novel routing framework that recasts the routing task as a matching process of sifting similar queries from historical logs, effectively mitigating the memorization trap. To enhance matching accuracy, we introduce a query capability deconstruction method that decouples linguistic surface forms from task-intrinsic requirements, directing matching toward capability dimensions to ground decisions in essential task attributes. Furthermore, we develop CodaSet, a comprehensive benchmark for assessing routing generalization, where experimental results demonstrate that DecoR maintains superior accuracy while substantially lowering inference costs across both in-distribution and OOD settings. All the codes and data are available at https://github.com/lvbotenbest/DecoR.

- **核心问题**：在部署大语言模型（LLMs）时，用户查询的复杂度差异很大，并非所有任务都需要调用大规模、高成本的模型。现有的大语言模型路由方法（如RouterDC、KNN-Router等）主要依赖于基于查询表面特征（如语义嵌入）直接映射到候选模型，这种“黑盒”映射机制容易陷入**记忆陷阱（memorization trap）**——路由器过度依赖训练数据中的表面模式，而非理解底层能力需求，导致在分布外（OOD）数据上泛化能力急剧下降。实验显示，现有方法在OOD场景下性能甚至不如随机选择基线，而成本却很高。此外，这些方法通常端到端训练，模型池更新后需要昂贵重训练。

- **整体含义**：为了解决上述问题，论文提出了一种全新的路由框架**DecoR**，将路由任务转化为**基于历史日志的匹配过程**：通过分析当前查询的能力需求，在历史日志中寻找最相似的查询及其模型表现，据此做出路由决策。这种方法从根本上避免了记忆陷阱，并支持通过简单更新历史日志实现系统迭代，无需重训练模型。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- 将路由问题重新定义为**匹配过程**：根据当前查询的能力需求，从历史日志中提取最相关的查询-模型经验，利用这些经验中模型的绩效和成本来估计最优模型。
- 通过**查询能力解构**，将语言表面形式与任务内在需求分离，使匹配基于能力维度（技能、知识、难度），而非表面语义。

### 关键技术细节（三个主要阶段）

**阶段1：查询解构阶段（Query Deconstruction Stage）**
- 使用一个**查询解构器（Query Deconstructor）**，将原始查询 `q` 转化为结构化**能力档案（Capability Profile）** `p = {S, K, D}`：
  - **S（技能集）**：原子功能操作列表（如信息抽取、摘要），附带解释。
  - **K（知识领域）**：所需的领域专业知识（如医学、法律），附带解释。
  - **D（难度）**：四个等级（d0-d3），附带解释。
- 训练方式：通过GPT-5合成数据，经三位CS博士生专家审查，然后进行**监督微调（SFT）**。

**阶段2：分层日志筛选阶段（Hierarchical Log-Sifting Stage）**
- 先离线用查询解构器为历史日志扩充能力档案：`H = {(qi, m_ij, r_ij, p_i)}`。
- 对新查询执行三个子阶段：
  - **子阶段A（字符级筛选）**：基于能力约束的Jaccard相似度（Skills和Knowledge）乘以难度校准权重 `w`（历史难度不低于当前时为1.0，否则每低一级减0.25）。保留分数 `Score_A ≥ τ`（默认τ=0.5）的日志。若无日志通过，则判定为OOD，直接进入决策阶段触发回退。
  - **子阶段B（精细排序）**：用BGE-M3模型对查询文本和其能力档案的拼接进行编码，计算余弦相似度，保留Top-k（默认k=3）个日志。
  - **子阶段C（日志对齐评估）**：使用**日志评估器（Log Evaluator）**进行长上下文推理，根据当前查询和能力档案与候选日志，输出一组有效代表日志标识符V。如果输出空集，也视为OOD。该模块通过**GRPO（组相对策略优化）**训练，奖励函数鼓励高召回并惩罚幻觉。

**阶段3：经验决策阶段（Empirical Decision Stage）**
- 如果是OOD（子阶段A未通过或子阶段C为空集），则回退到高性能默认模型（如DeepSeek-V3.1）。
- 如果V非空，则从历史日志中获取对应模型在V中的平均性能 `V_j` 和平均成本 `C_j`，进行线性归一化（双维度无量纲处理），然后通过平衡因子 `λ`（默认0.5）计算综合效用分数 `U_j = λ·V_norm_j + (1-λ)·C_norm_j`，选择效用最高的模型作为最终路由目标。

### 公式与算法
- `Sim_sk(q_u, q_i) = |Su∩Si|/|Su∪Si| + |Ku∩Ki|/|Ku∪Ki|`
- `w(D_i, D_u)` 难度匹配公式
- `Score_A = Sim_sk × w`
- 归一化：`V_norm_j = (V_j - min(V))/(max(V)-min(V)+ε)`；`C_norm_j = (max(C)-C_j)/(max(C)-min(C)+ε)`
- 效用：`U_j = λ·V_norm_j + (1-λ)·C_norm_j`

1. **DecoR在ID和OOD场景下均保持高精度**：平均性能接近最强单模型（如Qwen3-235B-A22B），但成本显著降低（例如ID平均成本2.1x vs 5.0x）。

5. **τ=0.5是默认最优阈值**。
6. **0.6B基模型足够**：更大规模模型带来增益极小，因此选择轻量级0.6B Qwen3作为基础。
