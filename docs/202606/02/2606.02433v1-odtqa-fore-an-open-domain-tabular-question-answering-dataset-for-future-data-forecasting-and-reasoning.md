## 摘要
大语言模型的快速发展显著推动了表格问答的进步，但大多数系统无法进行面向未来的数值预测。为填补这一空白，我们引入了一项新任务——面向未来数据预测与推理的开放域表格问答，并提出了首个覆盖时间序列预测和基于预测的推理场景的数据集，采用房地产数据。该任务在检索精确历史数据、克服大语言模型预测局限性以及为多样化查询标准化响应方面带来了挑战。为解决上述挑战，我们提出了TimeFore，一种基于大语言模型智能体的框架，将问题分解为三个协作角色：检索器自主生成SQL获取数据，预测器调用外部时间序列模型以提高准确性，分析器综合结果构建精确一致的最终答案。大量实验证明了我们TimeFore的有效性。

## Abstract
The rapid development of LLMs has significantly advanced tabular question answering, but most systems cannot perform future-oriented numerical prediction. To address this gap, we introduce a novel task, Open-Domain Tabular Question Answering for Future Data Forecasting and Reasoning, and propose the first dataset to cover time-series forecasting and forecast-based reasoning scenarios using real estate data. This task poses challenges in retrieving precise historical data, overcoming the forecasting limitations of LLMs, and standardizing responses for diverse queries. To solve the above challenges, we propose TimeFore, an LLM agent-based framework that decomposes the problem into three collaborative roles: a Retriever autonomously generates SQL to fetch data, a Forecaster invokes external time-series models for higher accuracy, and an Analyzer synthesizes the results to construct a precise and consistent final answer. Extensive experiments demonstrate the effectiveness of our TimeFore.

# 论文总结：ODTQA-FoRe：面向未来数据预测与推理的开放域表格问答数据集

## 1. 核心问题与整体含义（研究动机与背景）

当前基于大语言模型（LLM）的表格问答系统主要聚焦于静态历史数据的检索与推理，无法有效回答涉及未来数值预测的问题（如“下个月房价是多少？”或“哪个小区涨幅更大？”）。然而，用户在实际场景（如房地产投资、金融决策）中频繁需要这类未来导向的预测与推理。现有方法的局限性在于：

- LLM 训练于静态文本，缺乏针对动态时间序列数据的建模机制；
- 在开放域场景下，系统需自主从大型数据库中检索历史数据，而非依赖预定义表格；
- 问题类型多样（时间序列预测 vs. 基于预测的推理），要求系统能区分并生成标准化答案。

为此，论文定义了新任务 **ODTQA-FoRe（Open-Domain Tabular Question Answering for Future Data Forecasting and Reasoning）**，并构建了首个面向未来数据预测与推理的开放域表格问答数据集（基于房地产数据），填补了该领域的研究空白。

## 2. 方法论：TimeFore 框架

### 核心思想
将整个任务分解为三个协作角色（LLM Agent），分别处理数据检索、时间序列预测和答案合成，以弥补 LLM 在数值预测上的固有不足。

### 关键技术细节

- **Retriever（检索器）**：
  - **第一阶段**：使用 LLM（few-shot）将用户问题总结为规范的表格标题样式，尝试与数据库现有标题直接匹配。
  - **第二阶段**：若直接匹配失败，则使用 BM25 算法检索最相关的表格标题。
  - **SQL 生成**：基于检索到的表格标题和问题，通过 ICL（5个示例）生成 SQL 语句，并利用执行反馈循环（最多 25 次重试）确保 SQL 可执行。
- **Forecaster（预测器）**：
  - 将历史数据格式化为 `[项目名，年月，价格]` 的三元组列表（24个月窗口）。
  - 利用 LLM 的函数调用能力，调用外部专用时序模型：**TimesNet** 进行数据插补（处理稀疏缺失值），再使用 **TimeXer** 进行未来 12 个月的价格预测。
- **Analyzer（分析器）**：
  - 使用 **BERT 分类器** 将问题分为“时间序列预测”和“基于预测的推理”两类，分别对应不同的 ICL prompt。
  - 对于预测类问题，引入 **LLM 驱动的数值提取模块**（in-context learning + 5 示例），强制模型仅返回数值结果而非冗长解释。
  - 最终输出格式标准化：单项目问题直接输出数值，多项目问题输出带项目名和月份的列表。

### 算法流程（文字说明）
1. 用户问题输入 → Retriever 生成表格标题 → 生成 SQL → 执行并获取历史数据（三元组列表）。
2. 历史数据传给 Forecaster → 转换为 24 个月数值序列（缺失值用 `"-"` 占位）→ 调用 imputationThenPredictionTool（TimesNet 插补 + TimeXer 预测）→ 返回 12 个月预测值（三元组列表）。
3. 历史数据与预测数据传给 Analyzer → BERT 分类 → 选择对应 prompt → 结合 ICL 示例生成结构化答案 → 数值提取模块标准化输出。

## 3. 实验设计

### 数据集
- **ODTQA-FoRe**：基于 2022–2024 年中国 10 个城市的房地产交易记录，共 28,507 个 QA 对，288 张表（平均每表 845 行）；包含 8,042 个时序预测问题和 20,465 个推理问题；分为训练/验证/测试集（6:2:2 项目级别划分）。
- 时间分裂：2022-2023 为历史数据，2024 为未来标签（严格不参与推理）。
- 数据预处理：使用 TimesNet 对历史数据缺失值进行插补。

### Benchmark / 对比方法
- **Vanilla 基线**：BM25 检索表格 + 5-shot ICL 直接由 LLM 预测。
- 对比模型：Qwen3 30B A3B、Qwen3 Next 80B A3B Thinking、GPT-OSS 20B、GPT-OSS 120B、GLM 4.5 Air。
- 时序模型选择实验：对比 TimesNet、TimeMixer、TimeXer、WPMixer、AutoTimes、Time-MoE，以及通用 LLM（Qwen3 30B、GLM 4.5 Air），最终选用 TimeXer。

### 评估指标
- 时序预测：MSE、MAE、MRE。
- 推理任务：Accuracy（严格匹配）、Precision、Recall、F1（item-level）。

## 6. 主要结论与发现

1. **TimeFore 显著优于 Vanilla 基线**：在所有 5 个 LLM 上，TimeFore 在预测准确性和推理精确度上均大幅领先。
2. **预测是主要瓶颈**：消融实验表明，提供真实未来数据（GH+GF）使推理 Accuracy 提升约 44-49%，而提供真实表格或历史数据的改进很小（<2%）。说明时序预测质量是决定最终性能的关键。
3. **专用时序模型优于通用 LLM 直接预测**：表 2 显示 TimeXer 等模型 MSE/MAE/MRE 远低于 Qwen3 30B 和 GLM 4.5 Air（直接 ICL 预测）。
4. **LLM 协同外部工具策略有效**：通过函数调用将预测任务委托给专用模型，可显著提升整体系统能力。
5. **数值提取模块提高回答有效性**：去除该模块后，Qwen3 30B 的 Valid Completion Rate 下降 33%，说明 LLM 倾向于输出冗长文本而非简洁数值。

## 7. 优点

- **任务创新性**：首次定义并构建了面向未来数据预测的开放域表格问答任务，拓展了 LLM 在数值推理领域的应用边界。
- **数据集质量**：基于真实世界房地产数据，规模较大（28,507 QA 对），包含两种类型问题，具有明确的时间分裂和项目级划分，避免数据泄露。
- **框架设计的模块化与可扩展性**：Retriever-Forecaster-Analyzer 三阶段分离，各模块可独立改进（如替换更好的时序模型或检索策略）。
- **检索两阶段策略**：LLM 摘要 + BM25 退火，显著优于纯 BM25（F1 从 90.45% 提升至 97%+）。
- **Analyer 的分类 + 数值提取**：BERT 分类准确率接近 100%，数值提取模块有效标准化答案格式。
- **消融实验设计**：通过系统替换 golden 输出来定量分析各环节瓶颈，归因清晰。

## 8. 不足与局限

- **领域单一性**：数据集仅覆盖房地产领域，虽然框架设计为领域无关，但未在其他领域（金融、零售、气候）验证，泛化性存疑。
- **未纳入外部因素**：预测模型（TimeXer）仅基于历史价格，忽略了宏观经济、政策调整、环境因素等外生变量，在重大市场波动下可能失效。
- **模板生成数据局限性**：虽然经过 LLM 重写和人工评估，但底层仍基于 26 个种子模板，可能无法覆盖真实世界中所有复杂查询模式。
- **评估方式依赖 ground truth SQL**：未来数据通过 ground truth SQL 从保留库中获取标签，假设未来可精确查询，但实际应用场景中未来数据不可得。
- **计算资源需求较高**：TimeFore 依赖多个 LLM 调用和外部时序模型，推理成本较高（尤其 GPT-OSS 120B 需 8×A800）。
- **缺乏与其他开放式基准的直接比较**：由于任务是新提出的，未与现有 ODTQA 方法（如 RETQA、Open-WikiTable）进行 head-to-head 比较，但不影响其作为基线基准的价值。
