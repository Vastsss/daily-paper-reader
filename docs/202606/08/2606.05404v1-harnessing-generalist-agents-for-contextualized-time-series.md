## 摘要
时间序列通常嵌入在丰富的上下文中，这些上下文对于整体建模至关重要。此外，实际从业者通常需要端到端的工作流来分析时间动态，其中广泛研究的任务（如预测）只是更广泛解决方案循环中的一步。虽然通用AI智能体为复杂上下文下的此类工作流提供了有前景的接口，但它们主要运行在文本空间中，未能完全与结构化时间信号对齐。在这项工作中，我们提出了TimeClaw，一个面向时间序列的智能体框架，它为通用LLM智能体配备了时间序列原生的运行时支持，以实现情境化时间推理。TimeClaw集成了可执行的时间工具用于基于事实和可审计的分析、经验驱动的能力演化以创建可重用的分析例程，以及用于检索相关推理轨迹的情境多模态记忆。这些组件共同释放了利用上下文信息的开放式时间推理能力。在涵盖能源、金融、天气、交通及其他实际领域多样任务的多个基准上的广泛评估表明，TimeClaw的性能得到了提升。代码可在https://github.com/iDEA-iSAIL-Lab-UIUC/TimeClaw获取。

## Abstract
Time series are often embedded in rich contexts that are essential for holistic modeling. Moreover, real-world practitioners often require end-to-end workflows for analyzing temporal dynamics, where widely studied tasks such as forecasting are only one step in a broader solution loop. While generalist AI agents offer a promising interface for such workflows under complex contexts, they still operate primarily in textual spaces that are not fully aligned with structured temporal signals. In this work, we introduce TimeClaw, an agentic harness framework for time series that equips generalist LLM agents with the time series-native runtime support needed for contextualized temporal reasoning. TimeClaw integrates executable temporal tools for grounded and auditable analysis, experience-driven capability evolution for creating reusable analytical routines, and episodic multimodal memory for retrieving relevant reasoning traces. Together, these components unlock harnessed open-ended temporal reasoning with contextual information. Extensive evaluation on multiple benchmarks covering diverse tasks across energy, finance, weather, traffic, and other real-world domains demonstrates improved performance of TimeClaw. Code is available at https://github.com/iDEA-iSAIL-Lab-UIUC/TimeClaw.

### 1. 核心问题与整体含义（研究动机与背景）

**问题背景**：时间序列分析常依赖丰富的上下文信息（如文本描述、领域知识），现实世界从业者需要端到端的工作流，而不是单一的预测任务。然而，通用 LLM 智能体主要在文本空间操作，无法直接处理结构化时间信号，导致两种对齐问题：
- **数据类型不对齐**：将时间序列序列化为文本 token 会扭曲数值距离、位值结构等统计性质。
- **智能体过程不对齐**：现有智能体工作流以文本为中心，长数值序列会占用上下文窗口并分散智能体注意力，不利于可靠的时间推理。

**核心含义**：需要构建一个“时间序列原生”的智能体框架，使通用 LLM 智能体能够像处理文本一样自然、可靠地处理结构化时间序列及其上下文。

### 2. 方法论

#### 核心思想
提出 **TimeClaw**——一个时间序列原生智能体框架，包含三个关键组件：
- **可执行时间工具**（Executable Temporal Tools）
- **经验驱动能力演化**（Experience-Driven Capability Evolution）
- **情景多模态记忆**（Episodic Multimodal Memory）

#### 关键技术细节
1. **运行时环境与工作空间**：每个任务将时间序列以完整数值精度加载到本地工作空间 `W` 中，通过协议（如 MCP）访问。智能体通过与框架交互生成轨迹 `ζ = (a₁, o₁, ..., a_K, o_K, ŷ)`。
2. **时间工具**：提供原子操作（如 `list_channels`, `series_overview`, `channel_stats`, `compute_acf`, `detect_periodicity`, `find_peaks`, `arima_forecast` 等）。所有工具在数值精度下运行，返回紧凑的结构化观察结果。
3. **能力演化**：演化算子 `E` 识别已验证的、重复出现的子过程，将其抽象为自包含的可执行工具。每个候选工具在留出子集上验证，成功率超过阈值 γ 则加入工具集 `T`。
4. **多模态记忆**：
   - 每条记忆记录 `m = (τ, ϕ, ψ, ζ, ρ)`，包含任务、检索键、轨迹和可迁移解释。
   - 上下文键 `ϕ = Φ(c_τ)` 使用文本编码器（如 text-embedding-3-small）嵌入任务上下文。
   - 时间序列键 `ψ = Ψ(X_{1:T})` 是 20 维指纹（包括统计、趋势、自相关、频谱、变化点、多变量耦合等特征）。
   - 检索采用两阶段策略：先按文本余弦相似度选出候选池（如 20 个），再按归一化指纹 L2 距离排序返回 top‑k。

#### 算法流程（文字描述）
1. 给定任务 `τ = (q, X, Y, y^*, ℓ)`，将数值序列加载到工作空间。
2. **记忆检索**（如果启用）：从记忆库中检索 top‑k 相关轨迹，注入提示前缀。
3. **推理执行**：智能体通过循环执行操作（调用工具或推理步骤），收集证据链，最终输出 `ŷ`。
4. **训练阶段**（构建记忆库）：要求智能体生成 `<context_to_action>` 可迁移解释块，供测试时参考。
5. **能力演化**：在训练轨迹积累后，演化算子识别并验证可重用例程，加入工具集。

### 3. 实验设计

#### 数据集与基准
- **Context-is-Key (CiK)**：71 个上下文感知的时间预测任务，覆盖公共安全、气候、交通、能源、零售、机械、经济、合成动力学等领域。5 种上下文类型（时不变的、历史的、未来的、协变量的、因果的）。
- **TSRBench**：4125 个实例，覆盖感知、推理、预测、决策 4 个维度、15 个子任务，14 个领域。本文使用 20% 分层子集（825 个实例）。
- **TSAIA Finance**：150 个金融时间序列分析实例（VaR、夏普比率、市场 alpha/beta）。

#### 对比方法
- 传统时间序列模型：ARIMA、ETS、DLinear、PatchTST。
- 时间序列基础模型：Chronos-2、Lag-Llama、Moirai-Large。
- LLM/智能体用于时间序列：UniTime、Time-LLM、TS-Agent、TSci。
- 通用智能体流水线：直接提示、CoT、ReAct、自反思、多智能体反思。
- 开放模型：Qwen2.5/3、Gemma3、InternVL、ChatTS、TS-Reasoner、TimeOmni 等。

### 6. 主要结论与发现

- TimeClaw 在 **CiK** 上平均 RCRPS 为 0.1145，相比最强基线（多智能体反思 0.1294）提升 11.5%，且 token 消耗减少 43.6%。
- 在 **TSRBench** 上平均准确率达 49.76%，相比最佳基线（多智能体反思 42.96%）提升 15.8%，且使用最少 token（约 1.6 万 vs 最高 5.1 万）。
- 在 **TSAIA** 上准确率达 67.6%，远超金融专用智能体（最高 48.6%），提升 38.9%。
- 三个组件（工具、演化、记忆）均贡献显著，移除任意一个都导致性能下降。
- 记忆检索有效：检索大小 k=3 时效果最佳，更大或更小均略逊。
- TimeClaw 可与多种 LLM 骨干（GPT、Gemini、Claude）兼容，性能随骨干增强而提升。
