## 摘要
测试时微调（TTFT）是一种快速演进的范式，它通过检索相关序列、在其上更新模型，然后评估提示来使语言模型适应每个提示。然而，TTFT只有在速度快时才实用：选择和微调都发生在每次查询中，各自成为直接瓶颈。现有方法以速度换取质量：快速检索往往冗余，而更强的多样性感知选择则增加了高昂的每次查询成本。我们提出HullFT，一种几何方法来解决TTFT中的这两个瓶颈。给定一个查询，HullFT首先将查询嵌入表示为少数训练序列的稀疏凸组合，使用高效的免投影Frank-Wolfe优化。这产生了一个本质上相关且多样化的支持集。然后，我们通过几何整数化过程将分数凸权重转换为精确的整数多重集以进行微调。由此产生的多重性自然地创建了重复样本，我们利用梯度重用来摊销重复微调步骤中的前向后向计算。我们的实验表明，HullFT在质量-效率权衡上优于当前最先进的TTFT方法，以显著更低的总运行时间实现了更低的每字节比特数。

## Abstract
Test-time finetuning (TTFT) is a rapidly evolving paradigm that adapts a language model to each prompt by retrieving related sequences, updating the model on them, and then evaluating the prompt. However, TTFT is only practical if it is fast: selection and finetuning both happen per query, making each a direct bottleneck. Existing methods trade speed for quality: fast retrieval is often redundant, while stronger diversity-aware selection adds prohibitive per-query cost. We introduce HullFT, a geometric approach to TTFT that addresses both bottlenecks. Given a query, HullFT first represents the query embedding as a sparse convex combination of few training sequences, using efficient projection-free Frank-Wolfe optimization. This yields a support set that is inherently relevant and diverse. We then convert the fractional convex weights into an exact integer multiset for finetuning through a geometric integerization procedure. The resulting multiplicities naturally create repeated examples, which we exploit with Gradient Reuse to amortize forward-backward computation across repeated finetuning steps. Our experiments show that HullFT improves the quality-efficiency tradeoff over current state-of-the-art TTFT methods, achieving lower bits-per-byte at substantially lower total runtime.

## 1. 核心问题与整体含义（研究动机和背景）

测试时微调（Test-Time Finetuning, TTFT）是一种新兴的推理范式：对于每个输入查询（prompt），从大规模语料库中检索最相关的序列，对模型进行微调（通常只做一次梯度更新），然后再评估该查询。TTFT能显著提升模型性能，甚至让小模型在某些任务上匹敌大一个数量级的模型。然而，TTFT面临两个关键瓶颈：
- **检索阶段与微调阶段均需在每次查询时实时完成**，直接贡献于面向用户的延迟。
- 现有方法在速度与质量之间难以兼顾：快速检索（如简单k近邻）往往引入大量冗余样本，导致微调信号重复；而多样性感知的选择方法（如SIFT）能提升质量，但选择过程本身代价高昂，尤其是在小预算下。

因此，如何同时实现**快速且高质量**的测试时微调是该领域的核心挑战。本文提出一种基于凸几何的新方法HullFT，旨在统一解决检索的冗余性与微调的计算效率问题。

## 2. 方法论

### 核心思想
HullFT将数据选择建模为**近似Carathéodory问题**：将查询嵌入表示为候选池中少数序列的稀疏凸组合。该凸组合的支持集自然具备**相关性与多样性**——因为方向相似（冗余）的序列在逼近查询时贡献很小，而不同方向的序列会被优先选中。之后通过**几何整数化**将分数权重转换为整数多重集（确切包含N个样本），最后利用多次重复出现的样本实现**梯度重用**以加速微调。

### 关键技术细节（算法流程，用文字说明）

1. **稀疏凸选择（Frank–Wolfe支持集）**
   - 给定查询嵌入 \( q \) 和 \( K \) 个候选嵌入 \(\{p_i\}\)，在概率单纯形上求解最小化 \(\|q - \sum w_i p_i\|_2^2\) 问题，使用投影自由的Frank-Wolfe算法。
   - 每步选择与当前残差内积最大的候选点，进行精确线搜索，更新权重。算法自动忽略近重复点，仅保留稀疏支持集。
   - 停止条件：当残差平方小于阈值 \(\varepsilon\) 或支持集大小达到上限 \(m\) 时停止。

2. **几何整数化（Integerization）**
   - Frank-Wolfe输出分数权重 \(\tilde{w}_j\)，需转换为整数计数 \(c_j\) 使总数为 \(N\)，且均匀均值 \(\sum (c_j/N) s_j\) 尽量接近查询。
   - 三步过程：
     - **向下取整分配**：\(c_j = \lfloor N \tilde{w}_j \rfloor\)。
     - **贪婪填充**：将剩余未分配的单位逐一加到能使重构误差下降最多的支持点上。
     - **局部交换优化**：检查将一份数量从一点移至另一点是否能降低误差，进行两轮交换。
   - 最终每个支持点 \(s_j\) 出现 \(c_j\) 次，构成大小为 \(N\) 的多重集。

3. **梯度重用（Gradient Reuse）**
   - 在微调时，对同一序列的连续多次训练步（因为整数化产生了重复）**每 \(r\) 步才重新计算一次梯度**，中间步骤重复使用缓存的梯度。
   - 仅需 \(\lceil N/r \rceil\) 次前向-后向传播，而非 \(N\) 次。
   - 默认选择 \(r=2\)，在质量和速度间取得最佳平衡。

### 整体流程总结
给定查询 → 通过FAISS索引检索K个最近邻候选池 → 用Frank-Wolfe得到稀疏凸组合支持集 → 几何整数化生成N个训练样本的多重集 → 使用梯度重用对基础LLM进行微调 → 评估微调后模型在查询上的bits-per-byte。

- **HullFT在所有总运行时预算 ≤4秒 时严格优于kNN和SIFT**，以更低BPB%实现相同或更短的延迟。在严格预算下（如T=0.75s）优势最大（低6.4% BPB），在T=1.75s和2.0s时分别比最优基线低3.83%和3.44%。

- **批量效果广泛**：11/12个子集在T=1.75s时优于最优基线。

## 7. 优点

- **创新性**：首次将近似Carathéodory问题应用于TTFT的数据选择，通过凸几何统一相关性与多样性，理论完备（Carathéodory定理保证存在性）。
- **效率**：Frank-Wolfe是投影自由算法，仅需内积计算，迭代稀疏；整数化简单快速；梯度重用减少计算量。
- **鲁棒性**：对超参数（如ε、r）不敏感（支持集大小往往是实际约束），性能稳定。
- **解耦**：选择阶段独立于模型参数，可复用不同LLM的嵌入。
- **全面实验**：覆盖多领域、多消融、CPU/GPU比较、定性可视化，证据充分。

## 8. 不足与局限

- **候选池依赖**：HullFT在kNN预选池上运行，若上游检索召回率低，则凸组合表达力受限，最终质量受限于检索器能力。论文未直接分析检索召回对结果的影响。
- **固定嵌入空间**：选择在RoBERTa嵌入空间中进行，可能与下游LLM的损失景观不匹配。存在“嵌入-模型失配”风险，可能忽略对特定模型更有用的方向。
- **模型规模限制**：仅在GPT-2上实验，未验证在更大LLM（如7B、13B）上的实用性和速度。
- **实际部署延迟**：虽然总运行时间已降低，但平均仍需要1-2秒（加上检索约1.35秒），对于实时性要求极高的场景（如实时对话）可能仍有压力。
- **梯度重用风险**：连续多次使用相同梯度可能引入估计偏差，虽然实验显示影响轻微（0.64% BPB），但在某些子集上BPB略有上升（如DM Mathematics），需进一步验证其普适性。
- **实验设置局限**：每个子集固定150个查询，略少于SIFT/TTT-NN的全测试集（约1812点），但作者解释是为在有限算力下进行详细消融。另外，未报告多轮微调或不同学习率的效果。
