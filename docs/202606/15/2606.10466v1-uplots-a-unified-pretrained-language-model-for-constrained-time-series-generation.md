## 摘要
在时间序列生成中，现有方法通常为每个数据集手工设计或训练单独模型，这阻碍了可扩展性，且无法跨领域利用共享的时间结构。为解决这种碎片化问题，我们提出UPLOTS，一个统一的、基于提示引导的语言模型框架，用于跨不同领域的约束时间序列生成。UPLOTS不构建特定任务模型，而是利用由学习到的约束提示引导的单个预训练Transformer主干，实现具有精确模式控制的按需生成。一个关键创新在于动态的多数据集损失重新加权和提示到模式映射，这使得UPLOTS能够在训练期间内化多样化的时间结构，并在推理时有条件地生成它们。我们在四个真实世界基准和多种约束设置上评估UPLOTS，包括峰值周期、日历、负荷水平和波动模式。额外的保留约束组合与下游预测实验进一步表明，UPLOTS在原始峰值模式设置之外能够泛化，并在真实数据稀缺的情况下改进数据增强。我们的代码和基线可在匿名GitHub仓库获取：https://anonymous.4open.science/r/UPLOTS-6C36。

## Abstract
In time-series generation, existing approaches typically handcraft ortrain a separate model for each dataset, which hinders their scalability and fails to leverage shared temporal structures across domains. To address this fragmentation, we propose UPLOTS, a Unified, Prompt-guided Language model framework fOr constrained Time-Series Generation across diverse domains. Instead of building task-specific models, UPLOTS leverages a single pre-trained transformer backbone guided by learned constraint prompts, enabling on-demand generation with precise pattern control. One key innovation is our dynamic multi-dataset loss re-weighting and prompt-to-pattern mapping, which allows UPLOTS to internalize diverse temporal structures during training and conditionally generate them at inference. We evaluate UPLOTS on four real-world benchmarks and multiple constraint settings, including peak-period, calendar, load-level, and volatility patterns. Additional held-out constraint-combination and downstream forecasting experiments further demonstrate that UPLOTS generalizes beyond the original peak-pattern setting and improves data augmentation under scarce real-data regimes. Our code and baselines are available at anonymous github repo: https://anonymous.4open.science/r/UPLOTS-6C36.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：采用单个预训练LLM（GPT-2或LLaMA 3.2-1B）作为统一主干，通过“时间序列提示嵌入模块（TPEM）”将数据集特定的自然语言约束提示与时间序列输入融合，再结合动态加权训练策略，实现多数据集联合训练和推理时的条件生成。
- **关键技术细节**：
  - **时间序列提示嵌入模块（TPEM）**：
    - 将输入时间序列X通过卷积MLP投影到LLM的隐藏维度得到E_ts。
    - 将自然语言提示（如“Morning Peak Patterns in <DATASET>”）用分词器处理，通过冻结的词嵌入层得到提示嵌入E_pr。
    - 将E_ts和E_pr沿时间维度拼接，送入截断的预训练LLM（仅保留前两层的GPT-2或LLaMA 3.2-1B，仅训练LayerNorm和位置嵌入），输出再映射回原始特征空间，作为基础生成模型（如Diffusion-TS）的输入。
  - **动态加权训练策略**：
    - **课程感知损失调节器（CALM）**：维护每个数据集最近W步损失的滑动平均ℓ̄_i，根据损失相对大小计算静态课程权重w_i = s + (ℓ_max - ℓ̄_i)/(ℓ_max - ℓ_min)*(1-s)，在训练早期给困难数据集更低权重。
    - **滚动损失动态采样（RLDS）**：基于每个数据集滑动平均损失ℓ̄_i计算采样概率p_i = ℓ̄_i / Σℓ̄_j，使损失高的数据集被采样更多，直到某个加载器耗尽。
  - **基础生成模型**：采用Diffusion-TS，用Transformer预测原始信号而非噪声，损失联合时间域和频率域（L = ||x_t - ˆx_t||²_2 + λ ||F(x_t) - F(ˆx_t)||²_2），输出可分解为趋势、季节性和残差。

## 6. 主要结论与发现

- UPLOTS（无论GPT2还是llama3）在几乎所有数据集和指标上均显著优于单数据集SOTA（如Diffusion-TS）和多数据集基线TimeDP。例如，在Context-FID上，UPLOTS (GPT2)在ETTh_MP上为0.0243，而最佳单模Diffusion-TS为0.0478，TimeDP为0.2145；Discriminative Score上UPLOTS (GPT2)为0.0066，优于所有对比。

- 下游任务：在低数据率（1%-5%）下，UPLOTS增强数据显著提升预测性能；在高数据率下仍有微小改善。
- 超参数分析：隐藏维度64为最优；批次大小16以下不佳，32以上稳定；提示长度（单字符、短缩写、长句）影响不大，但语义丰富度比长度更重要（固定数字编码下降约5%）。

- **方法创新**：TPEM巧妙融合自然语言约束与时间序列；动态加权策略平衡多数据集训练，防止困难或噪声数据主导。
