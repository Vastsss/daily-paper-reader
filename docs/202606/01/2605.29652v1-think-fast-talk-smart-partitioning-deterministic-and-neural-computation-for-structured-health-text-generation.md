## 摘要
大型语言模型越来越多地被用于从结构化记录（如可穿戴时间序列、生物标志物、生命体征和护理管理日志）生成健康文本。对于重复性的健康输出，流畅性还不够：系统必须忠实于源数据，将解释性主张建立在现有证据上，遵循既定政策，输出机器可读结果，并且运行成本足够低以支持重复使用。我们探讨在结构化健康生成中，哪些职责应由确定性计算而非运行时LLM提示承担。我们引入了“快速思考，智能表达”这一睡眠健康洞察流程，其中确定性代码在调用一次有界LLM写入器之前执行重复分析。在280个用户夜间和六个模型上，与结构化零样本和少样本单次调用基线相比，该方法实现了更低的数值误差、更低的指令合规性误差和更低的端到端成本。层替换揭示了特定于合约的失败：LLM比较增加了数值误差，LLM排名降低了策略选择，LLM属性增加了无支持的因果语言，即使上游事实是确定性的，LLM生成的写入器接口也会重新引入错误。这些结果支持一个更广泛的设计原则：让代码负责重复分析，让LLM在有界接口内表达验证过的事实。

## Abstract
Large language models (LLMs) are increasingly being used to generate health text from structured records such as wearable time series, biomarkers, vitals, and care-management logs. For recurring health outputs, fluency is not enough: systems must remain faithful to source data, ground explanatory claims in available evidence, follow stated policies, emit machine-readable outputs, and run cheaply enough for repeated use. We ask which responsibilities in structured health generation should be deterministic computation rather than runtime LLM prompting.   We introduce Think Fast, Talk Smart, a sleep-health insight pipeline in which deterministic code performs recurring analysis before one bounded LLM writer call. Across 280 user-nights and six models, achieves lower numeric error, lower instruction-compliance error, and lower end-to-end cost than structured zero-shot and few-shot one-call baselines.   Layer replacement reveals contract-specific failures: LLM comparison raises numeric error, LLM ranking degrades policy selection, LLM attribution increases unsupported causal language, and an LLM-generated writer interface reintroduces errors even after upstream facts are deterministic. The results support a broader design rule: let code own recurring analysis, and let LLMs express verified facts within bounded interfaces.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
提出 **Think Fast, Talk Smart (TFTS)** 流水线，遵循“让代码负责重复分析（快速思考），让LLM在受限接口内表达已验证事实（智能表达）”的设计原则。整体架构将结构化健康文本生成分解为四个阶段（详见论文图1）：
1. **结构化输入**：原始可穿戴数据（睡眠、恢复、生理、行为信号）。
2. **确定性分析模块**：若干可独立验证的确定性层，包括：
   - 格式化数据
   - 数值比较（将测量值与基线对比）
   - 选择排名（根据规则选择最值得呈现的指标）
   - 归因门控（仅当证据超过阈值时才允许提及因果标签）
   - 压缩选定事实为紧凑的有界提示接口（handoff）
3. **有界LLM写入器**：一次LLM调用，将确定性接口中的已验证事实转化为符合schema约束的用户面向文本（标题、核心洞察、改进建议、属性标签、图表数据等）。LLM不能修改数值、不能添加新归因。
4. **确定性组装**：对LLM输出进行格式化和包装，确保符合机器可读schema。

### 关键技术细节
- 确定性代码与LLM之间的接口是一个**固定格式的JSON**，只包含已验证的数值、已筛选的归因标签、已排序的指标。
- 整个流水线只有一次LLM调用（写入器），对比基线（零样本/少样本）也是一次调用，但TFTS将分析逻辑置于LLM之外。
- 论文还设计了**层替换协议**：将TFTS中的某一个确定性层替换为由LLM生成的类型化中间产物（artifact），其余层保持确定性，以隔离分析特定失效模式。替换的条件包括：COMPARISON（数值比较）、RANKER（指标排名）、ATTRIBUTION（归因门控）、HANDOFF（写入器接口）。

## 6. 论文的主要结论与发现

1. **TFTS在误差-成本前沿占据优势**：在所有6个模型上，TFTS均达到低数值误差（平均NumErr 0.7%，最差2.0%）和低指令合规性误差（<3%），而零样本和少样本基线无法达到这一性能。尤其是少样本基线虽然改善格式和安全语言，但选择误差仍高达68.6%（参照表1）。

2. **层替换孤立出具体失效模式**：
   - **Replace Comparison**：数值误差上升至16.9%（LLM比较不可靠）。
   - **Replace Ranker**：选择误差高达65.8%（LLM排名不遵守策略规则）。
   - **Replace Attribution**：归因误差达24.5%（LLM引入无支持因果语言）。
   - **Replace Handoff**：即使上游事实均为确定性，替换写入器接口也导致NumErr升为4.8%、AttrErr升为10.7%，表明接口本身是可靠性关键。

3. **写入器接口是非显而易见的失效点**：LLM生成的中间接口即使包含已验证事实，也会在后续写入器调用中引入新错误，说明编译器级的事实传递设计比“让LLM自行选择表达方式”更可靠。

4. **设计规则**：对于重复分析的结构化健康系统，让确定性代码负责稳定可验证的分析（算术、选择策略、证据门控、写入接口），让LLM仅在受限接口内将验证事实转化为自然语言。

## 7. 优点

### 方法设计亮点
- **清晰的架构分离**：将“思考”与“表达”解耦，逻辑上鲁棒，易于审计和测试。
- **层替换协议**：不是简单地移除或削弱某部分prompt，而是要求LLM生成与确定性层等价的类型化中间产物，能精确测量每个分析环节的模型能力边界，是一个新颖的故障注入式分析方法。
- **低成本高可靠性**：即使是用最便宜的模型（GPT-5 nano，成本$0.0005/夜），TFTS也优于昂贵模型（Sonnet 4.6）的基线。

- 消融实验系统完整：既有全系统对比，又有逐层替换，直接回答了研究问题。

## 8. 不足与局限

### 应用场景局限
- 仅验证了睡眠健康一个领域。论文承认转移到实验室结果解释、慢性病管理、临床决策支持等任务时需要重新评估，尤其是开放推理的任务可能需要不同分工。
- 任务限定为固定schema的重复性输出，对于需要开放式推理、诊断、治疗规划的文本，TFTS的分区原则可能不适用。

### 评估局限
- 主要指标（SelErr, AttrErr, NumErr）均以与确定性参考系统的一致性为准，而非基于临床黄金标准或用户感知。例如，LLM选择一个与规则不同的指标可能对某些用户反而更个性化，但被记为错误。
- 未进行人类评估（有用性、自然度、信任度）。作者承认“临床最优性”未被验证，互补的人类评估是未来工作。

### 其他局限
- 数据集规模有限（20个用户，280个夜间），可能不能代表多样化人群。用户数量较少，可能存在个体偏差。
- 未探讨确定性代码的维护成本：规则需要人工定义和更新，当业务逻辑变化时需手动调整。
- 层替换实验虽然隔离了单个分析步骤，但替换后的结果仍然经过最终LLM写入器，存在交互效应；不过论文已分离写入器接口的影响（Replace Handoff）。
- 模型均为2025-2026年间版本，结论可能随模型能力快速演进而改变，特别是数值推理和策略遵循能力的提升。
