## 摘要
关键性能测量（KPM）预测对于5G及下一代电信网络的主动网络管理至关重要。然而，现有的机器学习方法在可扩展性和可解释性方面存在显著局限，限制了它们在实际部署中的有效性。我们提出了TelcoAgent，一个基于基础模型的框架，能够跨不同网络小区实现多KPM的准确、可扩展且可解释的预测，而无需针对特定站点进行训练。具体而言，该框架包含三个关键组件：(i) 一个自动化的三智能体流水线，直接从规范文档构建第三代合作伙伴计划（3GPP）知识图谱；(ii) 一个基于可扩展时间序列基础模型（TSFM）的预测流水线，以提供准确的零样本预测；以及(iii) 一个推理与解释流水线，提供可操作的、基于领域知识的诊断。使用来自美国网络运营商的三个月真实城市级5G KPM数据集进行评估，TelcoAgent在200个小区中对每个小区所有7个考虑的KPM实现了高预测精度，同时提供了可解释的洞察和可操作的指令来解决网络降级问题。

## Abstract
Key Performance Measurement (KPM) forecasting is essential for proactive network management of 5G and next-generation telecom networks. However, existing machine learning (ML) approaches face significant limitations in scalability and explainability, restricting their effectiveness in real-world deployments. We propose TelcoAgent, a foundation model-based framework that enables accurate, scalable, and explainable forecasting of multiple KPMs across diverse network cells without the need for site-specific training. Specifically, the framework comprises three key components: (i) an automated three-agent pipeline that constructs a 3rd Generation Partnership Project (3GPP) knowledge graph directly from specification documents, (ii) a scalable, time-series foundation model (TSFM)-based prediction pipeline to deliver accurate, zero-shot forecasting, and finally (iii) a reasoning and explanation pipeline that provides actionable, domain-grounded diagnostics. Evaluated using a 3-month, real-world, city-scale 5G KPM dataset from a U.S.-based network operator, TelcoAgent demonstrates high forecasting accuracy for all 7 considered KPMs per cell across 200 cells, while delivering explainable insights and actionable instructions to address network degradations.

  - **缺乏可解释性**：仅输出预测值，不能提供根因诊断和领域可解释的推理，运营商难以获得可操作的洞察。
- **背景**：FCC AI工作组指出KPM预测是网络监控、主动编排的核心能力。同时，时间序列基础模型（TSFM）展现出强大的零样本预测能力，大语言模型（LLM）也能结合知识图谱进行推理。但现有TSFM无法捕获电信领域特定的因果关系，也无法提供结构化解释。
- **整体含义**：本文提出的TelcoAgent框架，通过将TSFM与自动构建的3GPP知识图谱结合，实现了**可扩展、高精度、可解释**的多KPM预测，填补了“零样本预测”与“领域知识推理”之间的空白。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 2.1 整体架构
TelcoAgent由三个串联的流水线组成：
1. **知识图谱构建流水线**：从13份3GPP规范文档中自动提取结构化知识。
2. **预测流水线**：使用TSFM进行零样本多KPM联合预测。
3. **解释流水线**：基于ReAct范式，结合敏感性分析和知识图谱，生成诊断与推荐。

### 2.2 知识图谱构建（Pipeline 1）
- **输入**：3GPP TS 28.552, 28.554, 38.314, 38.211-38.215, TR 38.901, TS 38.300, 38.321, 38.322, 38.331等规范。
- **过程**：三个专用LLM智能体按序工作：
  - **抽取器（Extractor）**：将规范切分成节级块，提取“主体-谓词-客体”三元组，捕获KPM关系和因果链。
  - **对齐器（Aligner）**：将实体映射到规范本体，统一不一致术语为单一节点。
  - **评估器（Evaluator）**：为每个三元组分配置信度分数 \(q \in [0,1]\)，若 \(q < q_{TH}=0.9\) 则触发重新对齐循环。
- **输出**：本体模式包含KPM定义、物理量、公式、因果关系、环境类型等，形成有向因果图。

### 2.3 预测流水线（Pipeline 2）
- **模型**：使用TSFM（如Chronos-2）进行零样本预测。
- **输入**：滑动窗口历史观测矩阵 \(\mathbf{X} \in \mathbb{R}^{L \times C}\)，其中 \(L\) 为上下文长度（经调优选择：Chronos-2和Moirai用81天，MOMENT用22天），\(C=7\)为KPM通道数（RRC连接用户数、DL CQI、DL iBLER、DL rBLER、MAC DL吞吐量、PRB利用率、IP吞吐量）。
- **输出**：未来 \(H=168\)小时（7天）的联合预测 \(\hat{\mathbf{Y}} \in \mathbb{R}^{H \times C}\)。
- **关键**：Chronos-2和Moirai多通道联合预测，MOMENT独立预测每个通道。

### 2.4 解释流水线（Pipeline 3）
基于ReAct框架，逐步执行以下步骤：
1. **统计提取**：从预测轨迹计算均值、斜率、基线、百分比变化等关键指标。
2. **交叉通道敏感性分析**：使用PAX-TS（模型无关方法），通过系统变化源KPM并重新预测，计算敏感度矩阵 \(\mathbf{S} \in \mathbb{R}^{C \times C}\)。
3. **因果定向**：从3GPP知识图谱中检索有向因果路径，与敏感度结合，区分真正的因果关联与虚假相关。
4. **空间上下文**：集成OpenStreetMap（OSM）数据，捕捉地理影响（如高速公路、市区等）。
5. **根因分析**：联合敏感度分数、3GPP因果链、空间上下文，识别异常触发机制和相关RAN功能。
6. **行动建议**：针对具体RAN参数（如OLLA目标、HARQ限制、RRC不活动定时器）提出调整建议，并附证据链。
7. **数值保真校验**：自动交叉检查报告中所有数值（KPM描述符、敏感度分数）与流水线实际计算值是否一致，防止LLM幻觉。
