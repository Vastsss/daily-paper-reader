## 摘要
基于块的时间序列基础模型（TSFMs）存在上下文中毒问题：结构异常的块会吸引不成比例的注意力，并悄无声息地降低零样本预测质量。我们提出在推理时通过优化输入上下文而非修改模型权重来提高TSFM准确性。我们提出GITCO（门控推理时上下文优化），一个轻量级的三组件框架：门控、路由器和评判器，它能够在不进行任何参数更新的情况下选择性地识别并抑制有害块。在53个GIFT-Eval数据集上对TimesFM 2.5进行K折交叉验证评估，GITCO在TimesFM 2.5上实现了平均+1.95%的MASE降低，同时捕获了89.9%的改进上限。我们引入了上下文敏感性分布作为TSFMs的一个新的可表征属性：从时间序列元特征到推理时上下文干预下预期准确性改进的映射，该映射由模型架构和数据的统计结构共同塑造。

## Abstract
Patch-based Time Series Foundation Models (TSFMs) suffer from context poisoning: structurally anomalous patches capture disproportionate attention and silently degrade zero-shot forecast quality. We propose improving TSFM accuracy at inference time by optimizing the input context rather than modifying model weights. We present GITCO (Gated Inference-Time Context Optimization), a lightweight three-component framework: Gate, Router, and Critic that selectively identifies and suppresses harmful patches without any parameter updates. Evaluated on TimesFM 2.5 across 53 GIFT-Eval datasets under K-fold cross-validation, GITCO achieves an average +1.95% MASE reduction on TimesFM 2.5 while capturing 89.9% of the improvement upper bound. We introduce context sensitivity profiles as a new characterizable property of TSFMs: the mapping from time series meta-features to expected accuracy improvement under inference-time context intervention, shaped jointly by model architecture and the statistical structure of the data.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：基于 patch 的时间序列基础模型（TSFM）在零样本预测中容易遭受“上下文毒化”（context poisoning）。当输入上下文窗口中某个 patch 包含异常信号（如波动爆发、水平漂移、虚假季节性）时，该 patch 会吸引过多注意力，导致预测质量下降，即使其余上下文正常。
- **整体含义**：现有方法主要聚焦于修改模型本身（扩展预训练、架构调整、微调），而本文则开辟了“输入上下文优化”这一新范式：在不更新模型参数的前提下，通过在推理时选择性抑制有害 patch 来提升零样本预测精度。这类似于 LLM 中通过链式推理或自洽性改进输出，而非修改权重。

### 2. 论文提出的方法论

- **核心思想**：GITCO（Gated Inference-Time Context Optimization）是一个轻量级的三组件推理时包装器，作用于任何冻结的基于 patch 的 TSFM。它通过门控决定是否干预、路由选择专家评估器、评估器定位最有害 patch 并执行平滑，从而在不改变模型参数的情况下优化输入上下文。
- **关键技术细节**：
    - **Gate（门控）**：基于输入元特征 \(\phi(X)\) 的二元分类器，输出是否干预（\(g \in \{0,1\}\)）。采用精度优先的不对称损失函数设计，因为错误干预（假阳性）造成的平均性能下降幅度 \(|\mu^-|\) 超过正确干预（真阳性）的平均提升 \(\mu^+\)。在 TimesFM 2.5 上，决策边界主要由频域特征（季节性强弱、谱熵）驱动。
    - **Router（路由器）**：当门控决定干预时，路由器根据元特征选择三个专家评估器之一：`ShapeProbe`（CNN 基）、`StatProbe`（统计基）、`UniProbe`（通用基）。在 TimesFM 2.5 上，路由逻辑按波动性分裂：低变异系数且异常点密度低时选 UniProbe，低变异系数但异常点密度高时选 StatProbe，高变异系数时选 ShapeProbe。虽然路由准确率低（33.3%），但作为“遗憾最小化”工具，由于第二位候选常能提供相近收益，实际捕获了 89.9% 的理论上限。
    - **Critic（评估器）**：选中的评估器对每个 patch 分配一个破坏概率，定位得分最高的 patch \(\hat{i} = \arg\max c_i\)，然后对该 patch 应用 5 点简单移动平均（SMA）进行局部平滑，生成修正后的上下文。SMA 作为高效启发式，降噪效果主要依赖于 patch 的精确定位而非滤波器的复杂性。
- **算法流程**：输入上下文 \(X\) → 计算元特征 → Gate 判断是否干预（否：返回基线预测）→ Router 选择 Critic → Critic 评分所有 patch → 选择最高分 patch → SMA 平滑该 patch → 将修正后上下文送入冻结 TSFM 产生最终预测。

### 3. 实验设计

- **数据集**：53 个 GIFT-Eval 基准数据集（包含亚小时、小时、日、周/月等多个频率波段，涵盖经济、交通、天气等领域）。
- **基准模型**：TimesFM 2.5 和 Chronos2，均使用冻结、零样本设置。
- **对比方法**：主实验对比基线（无干预）；消融实验包括：
    - Always Intervene（始终干预）
    - Gate Only（仅有门控，默认使用固定 Critic）
    - Router Only（仅有路由，始终干预）
    - GITCO（门控+路由完整系统）
- **评估流程**：采用 K=11 折交叉验证，所有结果均为样本外预测。主要指标：MASE 降低百分比、捕获改进比（CIR）、Gate 精度/召回率。

### 6. 论文的主要结论与发现

- **主要结论**：GITCO 能有效提升冻结 TSFM 的零样本预测精度，在 TimesFM 2.5 上平均 MASE 降低 1.95%（绝对 MASE 改善 +1.032），捕获理论改进上限的 89.9%。精度优先的门控至关重要（78.0% 精度，57.6% 召回率），错误干预的代价远大于正确干预的收益。
- **关键发现**：
    - **上下文敏感性分布（\(\Phi_M\)）**：不同架构的 TSFM 具有不同的可学习性。TimesFM 2.5 的上下文改进性可由频域和波动性等元特征预测；而 Chronos2 的改进信号虽然真实存在（Oracle 分析显示 12–24 个数据集可改善），但无法从同一元特征集学出门控边界。这说明上下文干预的效果是模型架构和数据结构的联合属性。
    - **门控优先原则**：门控是系统价值的核心，路由器带来的增益是条件性的——只有门控安全，路由器才能有效。

### 7. 优点

- **创新性**：首次提出推理时上下文优化框架用于 TSFM，类比 LLM 的推理时计算扩展，开拓了新方向。
- **轻量可部署**：无需修改模型权重或参数更新，仅需在推理时添加轻量组件，适合生产环境（权重冻结、计算受限）。
- **设计精巧**：Gate 精度优先策略（不对称损失）有效避免破坏性干预；Router 作为遗憾最小化器，容忍选择错误；Critic 聚焦单个 patch 定位，简单高效。
- **引入新属性**：上下文敏感性分布（\(\Phi_M\)）为理解不同 TSFM 架构的优化潜力提供了新视角。
- **实验严谨**：K 折交叉验证、样本外评估、多种指标（CIR、精度/召回、总 MASE 变化），消融实验清晰。

### 8. 不足与局限

- **架构覆盖有限**：仅在 TimesFM 2.5 和 Chronos2 上评估，未测试其他流行 TSFM（如 PatchTST、Chronos1、MOIRAI、Lag-Llama 等）。Chronos2 的失败提示可能需不同元特征集，但未进一步探索。
- **干预空间固定**：仅使用 3 个专家 Critic 和 SMA 降噪，未探索更丰富的候选（如不同平滑窗口、masking、局部重新预测）。报告的 CIR 仅针对此固定词汇，更优干预可能改变上限。
- **分布偏移鲁棒性未验证**：实验基于 GIFT-Eval 静态分割，未考虑时间序列的分布漂移（概念漂移）。现实场景中上下文毒化模式可能随时间变化。
- **可重现性细节不足**：未提供算力消耗、具体 MLP 结构、训练/收敛细节。Gate 和 Router 的决策边界是通过滑动窗口跨数据集归纳的，但未清晰说明归纳方式（是否需要外部分类器训练？）。
- **局限性承认**：作者明确指出 Chronos2 欠佳的架构特定性，以及固定干预词汇的限制。Future work 应纳入更多模型和数据集，以及替代干预算子。
