## 摘要
大语言模型推理通常部署在静态资源假设下，模型无论运行环境如何都执行固定计算图。然而，现实云基础设施本质上是动态的，以资源可用性波动（如抢占式实例中断）和分层服务质量需求为特征。在这种不稳定的环境中，静态模型缺乏灵活性：它们要么在资源约束下崩溃，要么在冗余操作上浪费计算。为弥合这一差距，我们提出学习分配（L2A），一种用于资源自适应推理的端到端框架。与仅依赖于输入难度的先前方法不同，我们将推理建模为同时以输入和运行时资源预算本身为条件的约束分配问题。我们引入轻量级、预算条件和输入感知的门控网络，集成到大语言模型中。这些门控通过统一目标进行训练，该目标联合优化任务性能、逻辑一致性和资源成本，并沿三个方面匹配现实动态的表现：层跳跃应对内存和深度压力，头剪枝应对吞吐量争用，以及推理令牌减少应对延迟紧缩。这使得模型能够学习超越输入难度的预算感知策略：它根据实时资源动态自适应配置其计算足迹，在资源允许时最大化推理深度，而在预算紧张时强制执行严格节俭。单个L2A模型在Llama-3-8B和Qwen-3-4B上绘制了完整的计算-准确率帕累托前沿：在高达34%的实现层稀疏度下，它在GSM8K上保持与密集基线相差0.6%以内，在分布外任务上零样本下保持相同差距，而每个静态或启发式基线都需要单独调优的模型，并且在相当推理时间下仍下降5-10%。

## Abstract
Large Language Models (LLMs) inference is typically deployed under a static resource assumption, where models execute a fixed computational graph regardless of the runtime environment. However, real-world cloud infrastructure is inherently dynamic, characterized by fluctuating availability (e.g., spot instance preemption) and tiered Quality-of-Service requirements. In such volatile settings, static models are inflexible: they either crash under resource constraints or waste compute on redundant operations. To bridge this gap, we propose Learning to Allocate (L2A), an end-to-end framework for resource-adaptive inference. Unlike prior methods that condition only on input difficulty, we formulate inference as a constrained allocation problem conditioned on both the input and the runtime resource budget itself. We introduce lightweight, budget-conditioned and input-aware gating networks integrated into the LLM. These gates are trained via a unified objective that jointly optimizes task performance, logical consistency, and resource costs along three axes matching how real-world dynamics manifest: layer skipping for memory and depth pressure, head pruning for throughput contention, and reasoning-token reduction for latency tightening. This lets the model learn a budget-aware policy beyond input difficulty alone: it adaptively configures its computational footprint with respect to real-time resource dynamics, maximizing reasoning depth when resources permit while enforcing strict frugality when budgets tighten. A single L2A model traces the entire compute-accuracy Pareto frontier on Llama-3-8B and Qwen-3-4B: at up to 34% realized layer sparsity, it stays within 0.6% of the dense baseline on GSM8K, with the same gap holding zero-shot on out-of-distribution tasks, while every static or heuristic baseline requires a separately tuned model and still drops by 5-10% at comparable inference time.

## 1. 核心问题与整体含义（研究动机与背景）
- **问题**：当前大语言模型（LLM）推理通常部署在静态资源假设下，即模型无论输入难度和运行时环境，都执行固定计算图。然而，现实云基础设施存在动态性：资源波动（如抢占式实例中断）、分层 QoS 需求（如高级用户 vs. 免费用户）。静态模型无法弹性适应：资源紧张时要么因内存不足崩溃，要么延迟过高；资源充足时却浪费计算。
- **意义**：需要一种能够根据输入难度和实时资源预算动态分配计算资源的端到端框架，实现“预算感知”的自适应推理，使单个模型在资源波动场景下优雅降级，同时保持高性能。

## 2. 方法论
### 核心思想
- 将推理建模为**资源约束分配问题**，条件同时包括**输入隐藏状态**和**运行时预算信号** $b \in [0,1]$。
- 在冻结的 LLM 骨干中嵌入**轻量级、预算条件、输入感知的门控网络**，学习动态跳过层、剪枝注意力头、缩短推理token序列。
- 通过**统一目标函数**联合优化任务性能、逻辑一致性（知识蒸馏）和资源成本，涉及三个维度：层跳过（内存/深度）、头剪枝（吞吐量）、推理长度控制（延迟）。

### 关键技术细节
1. **预算信号**：标量 $b$ 通过嵌入 $e(b)$ 传入所有门控网络。训练时从分布 $D$ 采样；推理时通过实时系统信号（截止时间、队列长度、内存余量）标定得到。
2. **动态层跳过门控** $\phi_L$：对第 $l$ 层，输出 $\alpha_l = \sigma(MLP([h_{l-1}; e(b)]))$。训练时软缩放残差；推理时阈值二值化，$\alpha_l<0.5$ 则跳过整个层（$F_l$ 不执行）。
3. **动态注意力头剪枝门控** $\phi_H$：对每个头，类似结构输出 $\beta_{l,h}$，抑制不相关头。
4. **动态推理到答案过渡门控** $\phi_T$：在生成步骤 $t$ 输出 $\tau_t$，超过阈值时强制输出 `</think><answer>` 标记，缩短推理长度。
5. **参数高效适应**：冻结原始权重，注入 LoRA 低秩矩阵（秩 $r=16$），使骨干适应稀疏执行路径。
6. **统一损失函数**：
   $$L(b) = L_{ce} + \gamma L_{kd} + \lambda (C(b) - B(b)) + \lambda_T R_{tok}$$
   - $L_{ce}$：标准交叉熵，保持生成能力。
   - $L_{kd}$：KL 散度蒸馏（从密集教师），防止稀疏学生漂移。
   - 资源项：期望计算量 $C(b)$ 与目标预算 $B(b)$ 之差的惩罚。$C(b)$ 由层门控和头门控的加权和计算。
   - 推理长度代价 $R_{tok}$：缩短 `<think>` 段长度。
7. **推理时预算标定**：从实时信号（截止时间、中断时间、内存余量）映射到 $[0,1]$，取保守最小值 $b = \min\{b_{time}, b_{mem}, b_{queue}\}$。

## 6. 主要结论与发现
1. **L2A 达到优越的精度-效率帕累托前沿**：在 Llama-3-8B 和 Qwen-3-4B 上，单模型覆盖整个计算-精度曲线。在 34% 实际层稀疏度下，GSM8K 精度仅降 0.6%，而静态/启发式基线需单独调优且在相似时间下下降 5-10%。
2. **门控策略学到通用化的难度感知**：在简单输入上自动减少计算，在困难输入上保留全容量，且零样本泛化到 OOD 任务（HumanEval、BBH）。
3. **动态预算跟踪有效**：门控 α 随预算信号协同变化，训练稳定无门控崩溃。
4. **蒸馏是保持推理能力的关键**：去除蒸馏（γ=0）导致 GSM8K 从 35.8% 跌至 14.4%。
5. **联合层和头剪枝优于仅层剪枝**：相同延迟下，联合策略精度更高（GSM8K 35.8% vs 33.1%）。

## 7. 优点
- **端到端统一框架**：将动态稀疏性、参数高效微调、资源意识融入一个训练目标，无需多阶段调优。
- **预算条件门控**：首次显式将运行时资源预算作为门控条件，实现按需计算量调节。
- **三维稀疏控制**：层跳过、头剪枝、推理长度缩减三个自由度，协同发挥加速效果。
- **强大的泛化性**：单域训练即可泛化到 OOD 任务，且避免过拟合到特定难度模式。
- **训练稳定**：通过温度退火、蒸馏、预算惩罚避免门控崩溃，收敛行为良好。
- **实现简单**：基于 LoRA 适配，无特殊硬件需求，推理时直接二值化门控即可跳过计算。
