## 摘要
自2020年首次发布以来，Darts已成为广泛使用的开源Python时间序列分析库。近期，一系列基础模型声称在零样本预测中提升了准确性，有望实现从训练定制模型到利用预训练通用预测器的范式转变。然而，基础模型通常作为独立包发布，接口碎片化且与通用工具互操作性有限，使得在完整流程中进行联合评估和集成变得困难。在Darts中，我们开发了统一的\texttt{FoundationModel}类集合（Chronos-2、TimesFM 2.5、TiRex、PatchTST-FM），提供标准化、全周期的预测接口，且外部依赖最少，以便将基础模型集成到生态系统中。现有的Darts流程现在只需更改名称即可使用基础模型；新流程可将它们用于零样本或微调预测、不确定性估计和回测，结合数据处理与评估工具，这一切都在统一框架内完成。

## Abstract
Since its initial release in 2020, Darts has become a widely used open-source Python library for time series analysis. A series of foundation models have recently claimed accuracy improvements in zero-shot forecasting, promising a paradigm shift from training custom models to harnessing pre-trained general-purpose forecasters. Foundation models, however, are often released as isolated packages with fragmented interfaces and limited interoperability with common tooling, making joint evaluation and integration within complete pipelines difficult. In Darts, we developed a unified $\texttt{FoundationModel}$ class collection (Chronos-2, TimesFM 2.5, TiRex, PatchTST-FM) that provides standardized, full-cycle forecasting interfaces with minimal external dependencies for integrating foundation models into the ecosystem. Existing Darts pipelines can now use foundation models with only a name change; new pipelines can use them for zero-shot or fine-tuned forecasting, uncertainty estimation, and backtesting, combined with data processing and evaluation tooling, all within a unified framework.

## 1. 核心问题与整体含义（研究动机和背景）

近年来，时间序列基础模型（TSFMs）如 Chronos-2、TimesFM 2.5、TiRex、PatchTST-FM 等在零样本预测中展现出显著优势，有望替代传统定制模型。然而，这些模型通常以独立的软件包发布，接口碎片化、依赖冲突严重，且缺乏与通用时间序列工具（如数据预处理、评估、回测）的互操作性，导致用户难以对多个模型进行联合评估或将其集成到完整预测流程中。Darts 作为一个成熟的 Python 时间序列库，旨在通过提供统一的 FoundationModel 接口，将多种 TSFM 无缝接入其生态系统，降低使用门槛，推动零样本预测范式的普及。

## 2. 方法论：核心思想与关键技术细节

- **统一 API 设计**：新基类 `FoundationModel` 继承自 `TorchForecastingModel`，提供标准化的 `fit(series)` 和 `predict(n)` 方法，输入输出均为 `TimeSeries` 对象。即使是零样本预测也需调用 `fit`（仅做模型创建和检查），保证流程一致性。
- **后端与依赖**：使用 PyTorch Lightning 后端，支持早期停止、硬件加速；模型权重通过 HuggingFace Hub 的 safetensors 格式加载，但非强制。
- **核心功能支持**：
    - **协变量**：原生支持过去协变量和未来协变量，自动对齐，防止数据泄露。
    - **多序列预测**：支持对多个 `TimeSeries` 列表批量预测，自动分组和并行处理。
    - **历史预测与回测**：优化的 `historical_forecast` 方法，`retrain=False` 时批量处理历史时间点，提速明显；内置 `backtest` 和 `residuals` 方法。
    - **概率预测**：支持蒙特卡洛采样和直接参数预测两种模式，通过 `likelihood` 参数指定分布族（如分位数回归）。
    - **微调**：通过 `enable_finetuning` 参数控制：`False` 冻结参数做零样本；`True` 全参数微调；`dict` 可指定冻结/解冻特定层（支持通配符匹配）。该逻辑已提升到 `TorchForecastingModel` 基类，使所有 PyTorch 模型都能微调。
    - **可解释性**：内置 `ShapExplainer`，基于 Permutation SHAP 计算每个输入对预测的贡献。
- **模型提供方接入**：仅需两步：子类化 `PLForecastingModule` 定义前向传播，子类化 `FoundationModel` 实现 `_create_model` 返回模块实例。提供 `HuggingFaceConnector` 辅助下载。

## 6. 主要结论与发现

Darts 的统一 FoundationModel 框架成功将多个主流时间序列基础模型集成到同一生态系统中，用户只需更改模型名称即可切换模型，并能无缝使用 Darts 的数据处理、评估、回测等工具。验证表明 Darts 实现与原始实现性能高度一致（偏差 ≤1.3%），证实了接口设计的正确性和实用性。该框架显著降低了 TSFM 的使用和集成门槛，是推动零样本预测范式落地的关键基础设施。

## 7. 优点

- **统一接口、最小依赖**：仅需 `pip install darts[torch]`，无需额外安装各 TSFM 的孤立包。
- **功能全面**：原生支持协变量、多序列、概率预测、历史回测、微调、可解释性等，覆盖完整预测流程。
- **互操作性**：与 Darts 现有数百种模型和工具链无缝协作，支持集成、超参搜索、混合精度等。
- **易扩展**：模型提供方只需两步即可将新 TSFM 加入 Darts，省去开发通用工具的工作。
- **社区共建**：基于成熟的开源库，用户反馈和贡献可直接融入。

## 8. 不足与局限

- **支持的模型数量少**：启动时仅集成 4 个 TSFM（Chronos-2、TimesFM 2.5、TiRex、PatchTST-FM），相比 TimeCopilot（11 个）等库偏少。
- **缺乏参数高效微调**：尚未支持 LoRA 等 PEFT 方法，仅提供全参数或分层冻结/解冻。
- **实验验证范围窄**：仅在 fev-bench-mini 子集上验证，未涉及真实世界大规模、多变量、长时效预测场景；无消融实验证明微调或协变量功能的有效性。
- **可解释性方法有限**：仅采用 Permutation SHAP，未支持 Deep SHAP 或 Gradient SHAP。
- **潜在偏差风险**：验证中 TimesFM 2.5 有一个数据集“泄露”（可能因数据预处理差异），但作者已标注，整体结果可靠。
