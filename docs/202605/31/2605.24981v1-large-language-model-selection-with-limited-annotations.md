## 摘要
为给定任务选择大语言模型（LLM）需要比较众多强候选模型，然而标准评估依赖于对固定评估集进行昂贵的人工标注。为解决这一挑战，我们提出了SELECT-LLM，这是首个面向LLM的主动模型选择框架。SELECT-LLM旨在寻找一小组最具信息量的查询标注，以识别给定任务的最佳LLM。为此，我们引入了一种基于期望信息增益的查询选择规则，该规则通过候选模型输出之间的成对相似度计算得出。由于该规则仅使用生成的模型响应，因此SELECT-LLM可应用于所有候选模型，无需假设其架构或访问模型权重，因此同时适用于开放权重和黑盒LLM。我们在23个数据集、156个评估模型、多种任务家族以及多个文本评估指标上评估了SELECT-LLM。在所有实验中，SELECT-LLM在每个设置下均优于最强基线，最佳模型选择的标注成本降低高达81.8%，近最佳模型选择的标注成本降低高达84.78%。

## Abstract
Choosing a Large Language Model (LLM) for a given task requires comparing many strong candidates, yet standard evaluation relies on costly annotations over fixed evaluation sets. To address this challenge, we develop SELECT-LLM, the first framework for active model selection of LLMs. SELECT-LLM aims to find a small set of queries whose annotations are most informative for identifying the best LLM for a given task. To this end, we introduce a query selection rule based on expected information gain, computed from pairwise similarities between candidate model outputs. Because this rule only uses generated model responses, SELECT-LLM can be applied across candidate models without assumptions about their architecture or access to model weights. This makes it suitable for both open-weight and black-box LLMs. We evaluate SELECT-LLM across 23 datasets, 156 evaluated models, diverse task families, and multiple text evaluation metrics. Across all experiments, SELECT-LLM improves over the strongest baseline in every setting, with annotation cost reductions up to 81.8% for best model selection and up to 84.78% for near-best model selection.

### 论文总结：Large Language Model Selection with Limited Annotations (有限标注下的大语言模型选择)

#### 1. 研究动机与背景

- **核心问题**：如何以最小的标注成本，在众多候选的大语言模型（LLM）中，为特定任务或应用域高效、准确地选出**最佳模型**。
- **背景与挑战**：
    - **模型众多**：可用的大语言模型数量（包括开源和商业API）快速增长，使得模型选择变得复杂。
    - **性能依赖于上下文**：LLM的性能高度依赖具体任务和领域，不存在“通用最佳”模型。
    - **标准评估成本高昂**：传统的全量基准测试或人工评估需要的大量数据标注，成本昂贵且效率低下。
    - **现有方法局限**：现有的主动模型选择方法主要针对分类问题，而针对LLM生成场景的方法要么局限于两个模型的比较，要么只关注单模型的主动测试，没有解决从任意数量的候选模型中挑选最佳模型的问题。

#### 2. 方法论：SELECT-LLM

- **核心思想**：采用**主动学习**的策略，通过顺序地、智能地选择一小批最具信息量的查询（queries）进行人工标注，从而最大程度地区分和识别出最佳模型。其核心是一个基于**预期信息增益（Expected Information Gain）**的查询选择规则。
- **关键技术细节**：
    1.  **概率模型**：假设模型响应与参考答案的相似度（基于ROUGE、BLEU等指标）服从一个温度超参数τ控制的拉普拉斯分布。模型越好，其输出与参考答案越相似的概率越高。
    2.  **查询选择规则**：原始的信息增益计算在开放生成空间中难以追踪。作者通过两次数学逼近（二阶泰勒展开和一阶线性化），将信息增益最大化问题转化为一个**可追踪**的等价最小化问题。最终的选择规则为：
        > 选择能最小化候选模型对输出之间的**后验加权平均相似度**的查询。
        -   **直观解释**：如果当前两个被认为可能是最佳模型的候选，在一个查询上给出非常不同的输出，那么这个查询就很有信息量，能有效区分它们。反之，如果它们的输出相似，则信息量较少。
    3.  **算法流程**：
        -   初始化所有模型为等概率的最佳模型。
        -   在每一轮迭代中，从**未标注**的查询池中选出**最富信息**的查询（即能最小化后验加权平均相似度的查询）。
        -   获取该查询的人工标注（参考答案）。
        -   利用参考答案与每个模型输出的相似度（基于τ），通过贝叶斯更新所有候选模型的后验概率。
        -   重复上述步骤直到达到预算b。最终，后验概率最高的模型即为所选模型。
- **模型无关（Model-Agnostic）**：该方法仅依赖模型输出，无需访问模型权重、logits或内部状态，因此适用于开源和黑盒API模型。

#### 3. 实验设计

- **数据集与基准**：在 **23个** 广泛使用的LLM基准数据集上进行评估，涵盖推理（BBH, MMLU-Pro）、数学（GSM8K, MATH）、问答（NarrativeQA）、摘要（XSum, CNN/DailyMail）、翻译（WMT14）、情感分析（IMDb）、视觉问答（VQAv2）等任务，使用了 **6种** 不同的评估指标。
- **模型集合**：共评估了 **156个** 不同的LLM（包括不同规模、不同版本的多种模型，如GPT、Claude、Gemini、Llama、Qwen等），模型响应来自HELM, VHELM等现有基准框架。
- **对比方法**：由于缺乏现有的专门基线，作者将传统的主动学习方法（如随机采样、边界采样、最小协议、方差最小化、主动模型比较）适应到LLM选择场景作为强基线。
