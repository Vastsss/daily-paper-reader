## 摘要
模型内部状态编码了大语言模型（LLM）处理其训练数据的方式的丰富信息；然而，后训练数据工程主要依赖外部信号，忽略了模型内部状态中蕴含的丰富内在信号。我们提出了SAERL，一个用于LLM强化学习（RL）的数据工程框架。它使用稀疏自编码器（SAE）提取的模型内部状态，建模了三种内在数据属性：多样性、难度和质量。每种属性支撑一项具体的数据工程操作：用于批次多样性控制的SAE空间聚类与适度批次混合、用于从易到难课程排序的难度代理，以及用于数据过滤的质量探针。SAERL在Qwen2.5-Math-1.5B上相比原始GRPO平均准确率提高了3.00%，并以更少20%的训练步骤达到目标准确率，在不同模型规模和RL算法上均有一致提升。实验表明，SAE在不同模型家族和规模间有效迁移，作为轻量级且可复用的数据工程工具。这些结果证明，模型内部状态是后训练数据工程强大且实用的信号来源。

## Abstract
Model internals encode rich information about how a large language model (LLM) processes its training data; however, post-training data engineering largely relies on external signals and ignores rich intrinsic signals lying in model internals. We propose SAERL, a data engineering framework for LLM reinforcement learning (RL). It models three intrinsic data properties: diversity, difficulty, and quality, using model internals extracted with Sparse Autoencoder (SAE), an advanced mechanistic interpretability tool. Each property grounds a concrete data engineering operation: SAE-space clustering with moderate batch mixing for batch diversity control, a difficulty proxy for easy-to-hard curriculum ordering, and a quality probe for data filtering. SAERL improves average accuracy by 3.00% over vanilla GRPO and reaches target accuracy with 20% fewer training steps on Qwen2.5-Math-1.5B, with consistent gains across model scales and RL algorithms. Experiments show that SAE transfers effectively across model families and scales, serving as a lightweight and reusable data engineering tool. These results demonstrate that model internals are a powerful and practical source of signals for post-training data engineering.

### 1. 核心问题与整体含义（研究动机与背景）
- **核心问题**：当前大语言模型（LLM）的后训练（尤其是强化学习）高度依赖数据工程，但现有方法几乎完全依赖**外部信号**（如人类偏好、验证器结果、模型Rollout通过率等）来指导数据的选择、排序和分批。这些外部信号获取成本高，且忽略了训练过程中模型自身内部状态所蕴含的丰富信息。
- **研究动机**：探索模型内部状态（特别是通过稀疏自编码器SAE提取的精细特征）是否能作为有用且高效的信号，用于指导后训练数据工程，从而提升训练效率和模型性能。
- **整体含义**：该研究旨在提供一个新的视角，即利用模型内在的、可解释的信号（而非外部反馈）来驱动数据工程，从而开辟一条与现有外部信号方法互补的技术路径。

### 2. 方法论的详细描述（核心思想、技术细节）
- **核心思想**：提出**SAERL**框架，利用**稀疏自编码器（SAE）** 提取LLM的隐藏层激活，建模数据的三种内在属性：**多样性（Diversity）**、**难度（Difficulty）** 和**质量（Quality）**，并分别将它们映射到具体的数据工程操作。
- **关键技术细节与流程**：
    1.  **SAE表示提取**：
        - 对每个数据样本，分别提取其**提示（Prompt）和回答（Solution）** 的token级SAE激活。
        - 通过**均值池化**和**最大池化**对每个文本段的激活进行聚合，并与浅层元数据（如长度、TeX符号比例）拼接成一个**统一表示**用于后续操作。
    2.  **基于质量的数据过滤（Quality Probe）**：
        - 训练一个**线性分类器**，以SAE表示为输入，输出样本属于目标高质量分布的概率。
        - 根据概率阈值或Top-K排序，过滤掉低质量或偏离目标分布的噪声数据，得到更高质量的数据池。
    3.  **基于多样性的批次策略（Diversity-driven Batching）**：
        - **聚类**：使用MiniBatchKMeans算法对数据样本的SAE表示进行聚类（K=10）。
        - **适度批次混合**：将来自**不同聚类**、但**难度和长度相近**的两个批次进行“尾部交换”，即交换每个批次尾部的一小部分样本。这样既增加了批次内的多样性，又保留了大部分批次内的梯度一致性。
    4.  **基于难度的课程排序（Difficulty-driven Curriculum）**：
        - **难度代理**：在一个小的标注子集（3000个样本）上训练一个轻量级ElasticNet回归模型，以SAE表示为输入来预测样本难度。
        - **聚类内课程**：在每个聚类内部，根据校准后的难度分数进行升序排列（从易到难），形成局部的课程学习轨迹。
        - **全局课程**：按阶段（或轮次）从不同聚类中交错抽取批次，构成全局课程。在批次内应用前述的“适度批次混合”策略。
