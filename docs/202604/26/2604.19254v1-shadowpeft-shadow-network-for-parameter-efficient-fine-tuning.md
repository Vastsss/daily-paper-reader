## 摘要
参数高效微调 (PEFT) 通过仅训练少量任务特定参数并冻结预训练主干网络，降低了大语言模型 (LLM) 全参数微调的训练成本。然而，现有方法（如低秩自适应 LoRA）通过直接向单个权重插入独立的低秩扰动来实现自适应，导致了自适应的局部参数化。我们提出了 ShadowPEFT，这是一种中心化的 PEFT 框架，它通过深度共享的影子模块执行层级细化。在每个 Transformer 层，ShadowPEFT 维护一个并行的影子状态，并对其进行重复演化以获得逐渐丰富的隐藏状态。该设计将自适应从分布式的权重空间扰动转变为共享的层空间细化过程。由于影子模块与主干网络解耦，它可以跨深度重用、独立预训练，并可选择以分离模式部署，从而使边缘计算场景受益。在生成和理解基准测试上的实验表明，在可训练参数预算相当的情况下，ShadowPEFT 的表现达到或超过了 LoRA 和 DoRA。关于影子预训练、跨数据集迁移、参数缩放、推理延迟和系统级评估的进一步分析表明，中心化层空间自适应是传统低秩 PEFT 的一种具有竞争力且灵活的替代方案。

## Abstract
Parameter-efficient fine-tuning (PEFT) reduces the training cost of full-parameter fine-tuning for large language models (LLMs) by training only a small set of task-specific parameters while freezing the pretrained backbone. However, existing approaches, such as Low-Rank Adaptation (LoRA), achieve adaptation by inserting independent low-rank perturbations directly to individual weights, resulting in a local parameterization of adaptation. We propose ShadowPEFT, a centralized PEFT framework that instead performs layer-level refinement through a depth-shared shadow module. At each transformer layer, ShadowPEFT maintains a parallel shadow state and evolves it repeatedly for progressively richer hidden states. This design shifts adaptation from distributed weight-space perturbations to a shared layer-space refinement process. Since the shadow module is decoupled from the backbone, it can be reused across depth, independently pretrained, and optionally deployed in a detached mode, benefiting edge computing scenarios. Experiments on generation and understanding benchmarks show that ShadowPEFT matches or outperforms LoRA and DoRA under comparable trainable-parameter budgets. Additional analyses on shadow pretraining, cross-dataset transfer, parameter scaling, inference latency, and system-level evaluation suggest that centralized layer-space adaptation is a competitive and flexible alternative to conventional low-rank PEFT.

### 1. 核心问题与研究动机
*   **核心问题**：现有的 PEFT 方法（如 LoRA 及其变体）主要采用“局部参数化”方案，即在主干网络的各个线性层独立插入低秩扰动。这种方式导致适配过程在空间上是碎片化的，缺乏跨层的全局协调，且适配模块与主干网络结构深度耦合，难以独立迁移或在边缘设备上灵活部署。
*   **研究背景**：随着大模型（LLM）规模的扩大，全参数微调成本极高。虽然 LoRA 降低了成本，但其分布式权重更新的本质限制了适配器的模块化能力和跨规模复用性。

### 2. 方法论：中心化影子网络
ShadowPEFT 的核心思想是引入一个**深度共享的轻量级影子模块**，在 Transformer 的层级空间（而非权重空间）进行集中式适配。
*   **核心流程**：在每个 Transformer 层循环执行以下三个步骤：
    1.  **影子注入 (Shadow Injection)**：计算当前主干隐藏状态与影子状态之间的差异，通过一个低秩瓶颈层（$W_{down}, W_{up}$）将修正信号注入主干网络。
    2.  **主干编码 (Base Encoding)**：冻结的主干层处理注入修正后的隐藏状态。
    3.  **影子更新 (Shadow Update)**：采用类似 GRU 的门控残差机制，利用主干层的输出更新影子状态，使其随深度演化。
*   **关键技术细节**：
    *   **状态并行演化**：影子模块维护一个并行的隐藏状态 $s$，在跨层传递中不断积累任务相关信息。
    *   **可分离部署**：影子模块可以从主干网络中“分离”。在推理时，既可以采用“附着模式”（主干+影子）获得高性能，也可以采用“分离模式”（仅影子）实现极低延迟的边缘端推理。
    *   **影子预训练**：支持使用小型预训练模型（如 0.5B 模型）作为 8B 主干网络的影子，并通过摩尔-彭若斯广义逆（Moore-Penrose pseudo-inverse）进行初始化对齐。

### 6. 主要结论与发现
*   **性能优越**：在相同或更少的可训练参数预算下，ShadowPEFT 在多数任务上优于 LoRA 和 DoRA，尤其在 4B 和 8B 模型上优势更明显。

*   **参数扩展性**：与 DoRA 在参数增加时可能出现性能下降不同，ShadowPEFT 能有效利用更大的参数预算提升效果。

### 7. 优点与亮点
*   **架构创新**：打破了 LoRA 这种“权重扰动”的思维定式，转向“层空间细化”，实现了适配器的中心化管理。
*   **高度灵活性**：支持“附着/分离”两种部署模式，为云边协同提供了新思路（简单任务本地影子处理，复杂任务上传云端完整模型）。
*   **跨规模复用**：展示了小模型作为大模型适配器的潜力，具有良好的工程应用前景。

*   **模型覆盖面**：实验主要集中在 Qwen3 架构上，尚未在 Llama 或其他非 Transformer 架构（如 Mamba）上验证通用性。
*   **超参数敏感性**：影子模块的架构设计（如层数、宽度）需要根据主干网络进行调整，可能增加了一定的调参成本。
*   **算力限制**：受限于资源，未在 70B 以上的超大规模模型上进行验证。
