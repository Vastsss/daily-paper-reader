## 摘要
评估大语言模型（LLMs）在复杂、定量金融任务中的推理能力是一项关键且尚未解决的挑战。标准基准测试通常无法隔离智能体解析查询和编排计算的核心能力。为解决这一问题，我们提出了一种新颖的评估方法和基准，旨在严格衡量 LLM 智能体在金融时间序列分析中的推理能力。我们在一次大规模实证研究中应用了该方法，使用了我们的框架——时间序列增强生成（TSAG），其中 LLM 智能体将定量任务委托给可验证的外部工具。我们的基准包含 100 个金融问题，用于在评估工具选择准确性、忠实度和幻觉的指标上比较多个 SOTA 智能体（例如 GPT-4o、Llama 3、Qwen2）。结果表明，能力强的智能体可以实现近乎完美的工具使用准确性，且幻觉极少，验证了工具增强范式。我们的主要贡献是这一评估框架以及相应的关于智能体性能的实证见解，我们将其公开发布，以促进对可靠金融人工智能的标准化研究。

## Abstract
Evaluating the reasoning capabilities of Large Language Models (LLMs) for complex, quantitative financial tasks is a critical and unsolved challenge. Standard benchmarks often fail to isolate an agent's core ability to parse queries and orchestrate computations. To address this, we introduce a novel evaluation methodology and benchmark designed to rigorously measure an LLM agent's reasoning for financial time-series analysis. We apply this methodology in a large-scale empirical study using our framework, Time Series Augmented Generation (TSAG), where an LLM agent delegates quantitative tasks to verifiable, external tools. Our benchmark, consisting of 100 financial questions, is used to compare multiple SOTA agents (e.g., GPT-4o, Llama 3, Qwen2) on metrics assessing tool selection accuracy, faithfulness, and hallucination. The results demonstrate that capable agents can achieve near-perfect tool-use accuracy with minimal hallucination, validating the tool-augmented paradigm. Our primary contribution is this evaluation framework and the corresponding empirical insights into agent performance, which we release publicly to foster standardized research on reliable financial AI.

### 2. 论文提出的方法论：TSAG 框架
*   **核心思想**：采用“工具增强型 RAG”架构，将 LLM 作为编排者（Orchestrator），负责解析用户意图并调用预定义的 Python 定量工具。
*   **四层架构**：
    1.  **用户层**：前端接口（如 Telegram 机器人、Jupyter Notebook）。
    2.  **LLM 层**：基于 LangChain 的内核，连接不同的 LLM 智能体。
    3.  **工具层**：包含专门的 Python 函数（如计算波动率、相关性、模式识别等）。在评估阶段，使用“工具桩（Stubs）”返回预定义结果，以隔离评估智能体的推理逻辑。
    4.  **数据层**：提供时间序列数据库 API。
*   **工作流程**：
    1.  **解析与选择**：LLM 解析自然语言查询（NLQ），识别意图并提取参数。
    2.  **工具执行**：系统调用对应的 Python 函数进行可验证的计算。
    3.  **结果合成**：工具返回结构化数据。
    4.  **响应生成**：LLM 将结果转化为自然语言响应（NLR）。

### 6. 主要结论与发现
*   **能力分层**：GPT-4o 和 Qwen2 (7B) 表现最佳，在工具编排上达到了 **1.00 的完美匹配准确率**。
*   **模型规模效应**：较小的模型（如 Llama 3.2 3B 或 Qwen2.5 1.5B）在复杂推理和参数提取上表现较差，幻觉率显著升高。
*   **温度影响**：0.0 的温度设置能有效降低幻觉，更适合高风险的金融应用。
*   **延迟权衡**：本地模型（如 Qwen2 7B）响应速度快（约 2s），而 DeepSeek-V3 等 API 模型虽然推理能力强，但延迟较高（约 14s）。

### 7. 优点
*   **方法论创新**：提出了隔离“推理能力”与“执行结果”的评估范式，这对开发可靠的金融 AI 至关重要。
*   **实用性强**：TSAG 框架通过预定义工具解决了 LLM 算术能力差的痛点，确保了计算过程的可验证性。
*   **开源贡献**：公开发布了 100 个问题的金融基准测试集和评估框架，有助于行业标准化。

### 8. 不足与局限
*   **工具覆盖范围有限**：目前仅限于预定义的工具集，无法处理超出范围（Out-of-scope）的查询。
*   **缺乏动态性**：不支持动态生成 Python 代码，灵活性低于某些 Code-Interpreter 类的智能体。
*   **单步推理限制**：目前的 TSAG 工作流主要处理单步工具调用，无法处理需要多工具协作的复杂链式推理（如“找出波动率高且近期有正面新闻的资产”）。
*   **领域局限**：实验主要集中在加密货币领域，迁移到传统金融（股票、债券）需要重新适配工具和数据源。
