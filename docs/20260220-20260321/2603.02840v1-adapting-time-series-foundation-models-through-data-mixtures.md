## 摘要
时间序列基础模型（TSFMs）在零样本预测中变得越来越流行。然而，对于预训练集未完全覆盖的新时间序列领域，其性能可能会下降。因此，当从业者关注一个新领域并拥有一组相关数据集时，问题就出现了：如何最好地微调 TSFM 以提高零样本预测性能？解决此类问题的典型方法是在所有数据集上或分别在每个数据集上微调 LoRA 模块。在每个数据集上微调独立模块，可以通过为不同的时间序列上下文选择不同的每数据集模块组合，使 TSFM 专门化以适应不同类型的数据分布。然而，我们发现使用每数据集模块可能并非最优，因为一个时间序列数据集可能包含来自几种分布类型（即子领域）的数据。这可能是由于分布偏移或时间序列的不同维度具有不同的分布。因此，我们提出了 MixFT，它使用贝叶斯混合模型将数据重新划分为最能代表数据中存在的子领域的集合，并分别在这些集合上进行微调。这种数据的重新划分确保了每个集合更加同质，从而产生专注于特定子领域的微调模块。我们的实验表明，MixFT 的表现优于每数据集方法以及在所有数据上微调单个模块的方法。这表明，通过重新划分数据以代表子领域，我们可以更好地使 TSFM 专门化，从而提高零样本预测性能。

## Abstract
Time series foundation models (TSFMs) have become increasingly popular for zero-shot forecasting. However, for a new time series domain not fully covered by the pretraining set, performance can suffer. Therefore, when a practitioner cares about a new domain and has access to a set of related datasets, the question arises: how best to fine-tune a TSFM to improve zero-shot forecasting? A typical approach to this type of problem is to fine-tune a LoRA module on all datasets or separately on each dataset. Tuning a separate module on each dataset allows for the specialisation of the TSFM to different types of data distribution, by selecting differing combinations of per-dataset modules for different time series contexts. However, we find that, using per-dataset modules might not be optimal, since a time series dataset can contain data from several types of distributions, i.e. sub-domains. This can be due to the distribution shifting or having differing distributions for different dimensions of the time series. Hence, we propose MixFT which re-divides the data using Bayesian mixtures into sets that best represent the sub-domains present in the data, and fine-tunes separately on each of these sets. This re-division of the data ensures that each set is more homogeneous, leading to fine-tuned modules focused on specific sub-domains. Our experiments show that MixFT performs better than per-dataset methods and when fine-tuning a single module on all the data. This suggests that by re-partitioning the data to represent sub-domains we can better specialise TSFMs to improve zero-shot forecasting.

*   **研究动机**：传统微调方法通常在所有可用数据集上训练单个 LoRA 模块（共享微调），或为每个数据集训练独立模块（按数据集微调）。
*   **核心发现**：作者指出，按“数据集标签”划分数据并非最优，因为一个数据集中可能包含多种不同的分布（即“子领域”，如不同的季节性模式或波动性），且不同数据集之间可能共享某些子领域。
*   **目标**：探索如何通过重新划分数据，使微调后的模型能更好地捕捉潜在的子领域特征，从而提升零样本预测的泛化能力。

#### 2. 方法论 (MixFT)
作者提出了 **MixFT (Mixture Fine-Tuning)**，其核心思想是利用概率混合模型打破数据集的物理边界，按数据本质的分布特征进行聚类和微调。
*   **核心流程**：
    1.  **嵌入提取**：利用预训练的 TSFM 将所有微调数据映射到嵌入空间。
    2.  **子领域识别**：在嵌入空间中拟合一个**贝叶斯高斯混合模型 (Bayesian GMM)**。使用变分推理（Mean-field VI）来估计参数，并采用正态-逆对角-Wishart (NIDW) 分布作为先验，以减少过拟合。
    3.  **数据重划**：根据 GMM 的后验预测分布，将原始数据重新分配到 $K$ 个更具同质性的“子领域集合”中。
    4.  **专业化微调**：为每个子领域集合独立训练一个 LoRA 模块，形成一个 LoRA 专家库。
*   **推理机制**：在零样本预测时，模型首先通过 GMM 识别当前输入上下文（Context）最可能属于哪个子领域，然后调用对应的 LoRA 模块进行预测。

#### 6 主要结论与发现
*   **性能领先**：MixFT 在大多数数据集上的表现优于现有的按数据集微调方法和全量微调方法，平均排名最高。
*   **微调的难度**：在时间序列领域，许多传统的微调方法甚至不如不微调（Base），这反映了 TSFM 容易过拟合或发生灾难性遗忘的风险。
*   **子领域价值**：实验证明，同一个数据集中确实存在多个子领域（例如周期性切换），MixFT 能有效识别这些模式并分配给不同的专家模块。

#### 7. 优点
*   **打破边界**：创新性地提出按“子领域”而非“数据集标签”进行微调，解决了时间序列中常见的分布偏移问题。
*   **自动化与一致性**：利用贝叶斯框架，实现了微调阶段的数据划分与推理阶段的模块选择在逻辑上的一致性。
*   **鲁棒性**：通过引入 MixUp 数据增强和经验回放（Experience Replay），有效缓解了 PEFT 在时间序列任务中的过拟合问题。

#### 8. 不足与局限
*   **分布假设**：MixFT 假设微调数据和评估数据之间共享潜在的子领域。如果评估数据完全属于全新的分布，该方法可能失效。
*   **存储开销**：需要存储 $K$ 个 LoRA 模块，虽然 LoRA 参数量小，但在 $K$ 值较大时仍有一定内存压力。
*   **超参数敏感性**：虽然论文提供了选择 $K$ 值的方法，但 $K$ 的选择仍依赖于验证集的表现，且最优 $K$ 值可能随任务变化。
*   **单变量局限**：虽然实验涉及多变量数据，但方法论描述主要基于单变量视角，对复杂多变量间依赖关系的建模讨论较少。
