## 摘要
在实践中，大多数商业LLM提供商不会公开发布底层LLM架构的细节。然而，先前的研究表明，在有限API访问（即top-k logits和/或logit偏置函数）的情况下，可以恢复LLM的某些架构细节，如前馈网络的隐藏维度。或许作为对这些结果的回应，大多数商业LLM提供商已将其API限制为仅暴露每个解码token的单个logit，并且不再允许用户偏置logits。我们表明，即使在当前限制性API下，若干架构参数仍然可恢复。我们提出NightVision，一种攻击方法，利用限制性黑箱API访问来估计LLM的隐藏维度、深度和参数数量。算法上，NightVision依赖一种新颖的公共集提示技术，其中多个提示暴露同一组输出token的log概率；对这些结果进行频谱分析以推断隐藏维度。NightVision还利用端到端的首Token时间（TTFT）测量和估计的隐藏维度来估计深度和参数数量。我们在32个开源LLM上实证评估NightVision，所有模型的隐藏维度平均相对误差在23%以内（MoE模型为9%），对于超过30亿参数的模型，深度和参数数量的误差在53%以内。我们进行了大量消融实验，以展示这些精度如何随token预算和模型属性变化。总体而言，我们的结果表明，当前的LLM API并未充分限制以完全混淆其底层模型的架构细节。

## Abstract
In practice, most commercial LLM providers do not publicly release details of underlying LLM architectures. However, prior work has shown that given limited API access to an LLM (namely, top-$k$ logits and/or a logit bias function), one can recover certain architectural details of an LLM, such as the hidden dimension of the feed-forward network. Perhaps in response to these results, most commercial LLM providers have restricted their APIs to expose only the single logit for each decoded token, and they no longer give users the ability to bias logits. We show that even under current restrictive APIs, several architectural parameters are still recoverable. We present NightVision, an attack that uses restrictive black-box API access to estimate the hidden dimension, depth, and parameter count of an LLM. Algorithmically, NightVision relies on a novel common set prompting technique in which multiple prompts expose log probabilities for the same set of output tokens; a spectral analysis of these results is used to infer hidden dimension. NightVision additionally uses end-to-end time to first token (TTFT) measurements and the estimated hidden dimension to estimate depth and parameter count. We empirically evaluate NightVision on 32 open-source LLMs, recovering hidden dimension to within 23% average relative error across all models (9% on MoE models), and depth and parameter count to within 53% for models exceeding three billion parameters. We run extensive ablations to demonstrate how these accuracies scale with token budget and model properties. Overall, our results suggest that current LLM APIs are not sufficiently restricted to fully obfuscate the architectural details of their underlying models.

### 1. 核心问题与整体含义（研究动机和背景）

- **背景**：商业大型语言模型（LLM）提供商通常不公开其底层架构细节（如隐藏维度、层数、参数量）。先前研究（如Carlini et al., 2024；Finlayson et al., 2024）表明，在可以获取top-k logits或logit偏置函数的API访问下，可以通过谱分析恢复隐藏维度等架构属性。
- **问题**：为应对此类攻击，主流LLM API已大幅限制访问：仅返回每个解码token的单个log概率（single log-prob），且不再提供logit偏置功能。那么，在这种更严格的限制下，是否还能推断出模型的架构属性？
- **意义**：该研究有助于评估当前API设计能否有效保护模型知识产权，并为安全研究、模型审计提供技术手段。结果表明，即使API极度受限，仍可通过新的侧信道（如时序）泄露关键架构信息。

### 2. 方法论：核心思想、关键技术与算法流程

论文提出**NightVision**攻击方法，包含两个阶段：

#### 阶段一：公共集提示（Common-Set Prompting）—— 推断隐藏维度 \(d\)

- **目的**：在仅能获取单个采样token的log概率的情况下，构造一个可用于谱分析的log概率矩阵。
- **技术细节**：
  1. **生成高熵提示**：使用自动搜索生成大量难样本（如随机Unicode字符序列、分散分布的高温采样），使下一个token分布尽可能均匀。
  2. **采样与公共集构造**：对每个提示 \(S_q\) 采样 \(N\) 次，记录采样到的token及对应log概率 \(y_q^{(i)}\)。设 \(T_q\) 为每个提示下观察到的唯一token集合。定义公共集 \(C = \cap_q T_q\)，即所有提示都至少采样到一次的token集合。
  3. **矩阵构建与NaN剪枝**：构建稀疏矩阵 \(\hat{Y}_{full}\)，对缺失项NaN进行贪心剪枝（逐行/列移除NaN最多的行或列），得到稠密子矩阵 \(Y \in \mathbb{R}^{D' \times |C|}\)。
  4. **谱估计**：计算Gram矩阵 \(G = YY^\top\) 的特征值 \(\lambda_1 \ge \lambda_2 \ge \cdots\)。应用arctan变换 \(a_i = \arctan(\lambda_i)\)，并设计窗口斜率检测器定位鞍点和峰值，界定关注区间，再通过最大负二阶差分找到转折点作为估计 \(\hat{d}\)。最后上取整到128的倍数（常见模型隐藏维度对齐）。

#### 阶段二：时序攻击（Timing Attack）—— 推断层数 \(L\) 和参数量 \(P\)

- **核心思想**：LLM推理中，预填充（prefill）阶段的计算量随序列长度 \(\ell\) 以 \(O(L \ell^2 d)\) 比例增长。通过测量端到端的首Token时间（TTFT），并利用已知的参考模型拟合时序缩放关系，可以分离出 \(L\)。
- **技术流程**：
  1. **时序收集**：对目标模型和参考模型，设计长输入短输出的提示集合，使得预填充占主导。测量不同序列长度 \(\ell_j\) 下的TTFT \(T_{i,j}\)。
  2. **深度估计**：假设缩放关系 \(T_{i,j} \approx \beta L_i \ell_j^2 d_i + \alpha\)。在参考集上拟合 \(\beta, \alpha\)，然后代入目标模型的估计 \(\hat{d}\) 求解 \(L\)。考虑跨平台差异时，引入平台转移因子 \(k = \gamma_{\text{tgt}} / \gamma_{\text{ref}}\) 进行校准。
  3. **参数量估计**：假设目标模型为LLaMA风格的解码器（GQA、SwiGLU、未绑定embedding），根据公式：
     \[
     \hat{P} = 2 \hat{V} \hat{d} + \hat{L} \left(2 + 2\frac{n_{kv}}{n_{\text{heads}}} + 3\frac{d_{\text{ffn}}}{\hat{d}}\right) \hat{d}^2 + (2\hat{L}+1)\hat{d}
     \]
     估计总参数量，其中默认超参数取自LLaMA-3 8B（\(n_{kv}/n_{\text{heads}}=1/4, d_{\text{ffn}}/\hat{d}=3.5\)）。

### 3. 实验设计

- **数据集/场景**：使用32个开源LLM（验证集），外加7个开发模型（共39个）。覆盖8个模型家族：Qwen 2.5/3/3.5、IBM Granite 3.0–4.0、Meta Llama 3.1/3.2、AI2 OLMo 2/3/OLMoE、HuggingFace SmolLM2/3。参数范围135M–30B，隐藏维度 \(d \in [576,4096]\)，层数 \(L \in [16,48]\)，涵盖密集Transformer、MoE、混合架构（含滑动窗口注意力、Gated DeltaNet线性注意力）。

### 6. 主要结论与发现

1. **隐藏维度可恢复**：在32个模型中，平均相对误差23%（MoE模型仅9%），其中4/32模型精确恢复，12/32误差小于10%。随着token预算增加，误差大致单调下降。
2. **深度与参数量可联合估计**：对于≥3B参数的模型，深度和参数量的平均相对误差约53%。对于大型稠密模型（如Qwen2.5-7B、OLMo-3-7B），误差可低至16–20%。
3. **开销显著高于先验方法**：达到相同隐藏维度精度（≤30%相对误差），NightVision消耗的token是Carlini方法的12–190倍（中位数约1.7个数量级），反映了在弱威胁模型下的固有成本。
4. **时序侧信道泄露关键信息**：预填充主导的TTFT与 \(L \ell^2 d\) 呈线性关系，可用于推断深度和参数量。模型族内部一致性较好，但跨族（如Qwen 3.5）需单独校准。
5. **当前API不足以完全隐藏架构**：即使移除logit偏置和top-k logits，结合时序信息仍能恢复重要架构参数。

### 7. 优点

- **方法创新**：提出“公共集提示”技术，在极其受限的API（单log概率）下实现了隐藏维度的谱推断；同时开创性地将时序侧信道用于层数和参数量联合估计，拓展了黑盒攻击的维度。
- **理论支撑**：对公共集样本复杂度给出了形式化界（Theorem 1及非均匀推广Corollary 1），基于负相协（NA）的Chernoff界，为算法设计提供理论保证。
- **实验全面**：涵盖32个开源模型、多种架构（dense、MoE、混合注意力）和参数规模（135M–30B），并进行了跨平台校准和消融分析，结论稳健。
- **实用价值**：明确指出当前API设计的安全漏洞，为后代API的防御（如隐蔽时序、引入随机延迟）提供依据。

### 8. 不足与局限

- **成本与误差折中**：隐藏维度恢复需要极大token预算（\(10^{11} \sim 10^{12}\) tokens），对小模型成本相对更高，且误差仍可能较大（某些模型误差达49–64%）。
- **方法依赖高熵提示搜索**：自动寻找高效提示需要额外开销，且可能对某些模型（即分布极度陡峭的模型）效果不佳。
- **时序攻击的适用性限制**：仅对预填充主导的长上下文提示有效；对于<3B参数的小模型，预填充不占主导，误差超100%；对于混合架构（如Qwen 3.5的DeltaNet），需知道模型族才能正确缩放，否则误差较大。
- **架构假设**：参数量公式假设LLaMA风格（GQA、SwiGLU、未绑定embedding），对于不同结构（如不同FFN扩展比、不同注意力头配置）需默认假设，可能会引入误差。
- **未全面覆盖所有新兴架构**：未测试状态空间模型、纯线性注意力模型等，这些模型的计算模式可能完全改变时序缩放关系。
- **API设置理想化**：假设每次API调用返回稳定的log概率且无随机延迟；实际生产API可能加入缓存、批处理或噪声，影响有效性和准确性。
