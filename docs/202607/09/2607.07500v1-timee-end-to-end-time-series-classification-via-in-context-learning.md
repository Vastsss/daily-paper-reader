## 摘要
时间序列分类（TSC）目前主要采用两阶段范式：首先训练特征编码器（要么在目标数据集上从头训练，要么通过在大语料库上预训练），然后在其之上拟合特定任务的分类器。尽管有效，但这种解耦方式使得表示学习独立于分类目标进行优化，需要针对每个数据集进行训练，并且阻止模型在推理过程中利用标签信息。我们提出了TimEE，一个4.5M参数的基础模型，通过上下文学习实现端到端TSC。给定一个带标签的支持集和一个查询时间序列，TimEE在单次前向传播中直接输出预测的类别分布，无需针对每个数据集进行训练。遵循先验数据拟合网络（PFN）框架，TimEE完全在合成TSC任务上进行元训练，其中每个任务包含由生成过程中结构化分布偏移导致的不同类别身份的时间序列。尽管在预训练期间未见过真实时间序列，但在UCR基准测试中，TimEE在所有比较方法（包括基础模型和监督深度学习基线）中排名第一（ROC AUC）和第三（准确率）。据我们所知，TimEE是第一个在UCR基准测试中达到最先进性能的纯合成预训练模型。这些结果确立了具有合成先验的端到端ICL作为TSC领域一个引人注目且尚未充分探索的方向，其改进方向自然包括规模扩展、先验设计以及更丰富的生成机制。代码公开于 http://github.com/automl/timee。

## Abstract
Time series classification (TSC) is dominated by a two-stage paradigm: train a feature encoder -- either from scratch on the target dataset or via pretraining on large corpora -- and then fit a task-specific classifier on top. While effective, this decoupling optimizes representation learning independently of the classification objective, requires per-dataset training, and prevents the model from exploiting label information during inference. We introduce TimEE, a 4.5M-parameter foundation model for end-to-end TSC via in-context learning. Given a labeled support set and a query time series, TimEE directly outputs a predicted class distribution in a single forward pass with no per-dataset training required. Following the prior-data fitted network (PFN) framework, TimEE is meta-trained exclusively on synthetic TSC tasks, where each task contains time series with distinct class identities arising from structured distributional shifts in the generative process. Despite seeing no real time series during pre-training, TimEE ranks first in ROC AUC (and third on accuracy) on the UCR benchmark among all compared methods, which include both foundation models and supervised deep learning baselines. To our knowledge, TimEE is the first purely synthetic-pretrained model to reach state-of-the-art performance on the UCR benchmark. These results establish end-to-end ICL with synthetic priors as a compelling, largely unexplored direction for TSC, with scaling, prior design, and richer generation mechanisms as natural avenues for improvement. Code is publicly available at http://github.com/automl/timee.

## 1. 核心问题与整体含义（研究动机和背景）

- **当前TSC主流范式**：两阶段范式——先训练特征编码器（从头训练或在大语料库上预训练），再在其上拟合任务特定分类器。经典方法如MiniRocket、Hydra、TS2Vec以及近期基础模型如Chronos-2、TiRex、MantisV2均遵循此设计。
- **结构性瓶颈**：
  - 编码器优化独立于分类目标，表示学习与分类目标任务解耦。
  - 每遇到一个新数据集必须重新拟合分类器。
  - 推理时编码器无法直接访问带标签的训练样本，无法产生针对类别结构的表示。
- **本文目标**：通过上下文学习（ICL）解决上述三个问题，实现端到端时间序列分类，无需每数据集训练，且能利用标签信息。
- **关键挑战**：ICL元训练需要结构化的、标签有意义的合成任务，而现有时间序列合成方法要么生成无标签语料，要么任意分配类标签，均不满足要求。本文提出基于VARX的结构化合成先验来生成有意义的分类任务。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **ICL范式**：模型在推理时观察带标签的训练集（支持集），单次前向传播输出查询样本的类别分布，无需权重更新。
- **PFN框架**：遵循先验数据拟合网络（Prior-data Fitted Network），在合成任务上元训练，最小化查询点上的负对数似然。

### 2.2 合成数据生成（核心创新）
- **数据生成过程**：基于VARX（向量自回归过程带外生输入）模型生成多元时间序列：
  \[
  y_t = \sum_{k=1}^p A_k y_{t-k} + \sum_{j=0}^s B_j x_{t-j} + \epsilon_t,\quad \epsilon_t \sim N(0,\Sigma)
  \]
- **类别身份机制**（两种互补方式）：
  - **结构变化**：类别由依赖图结构（边存在与否）定义，反映变量之间相互作用的不同模式。
  - **信号变化**：依赖结构固定，类别由外生输入信号的统计特性（如扭曲速度）定义。
- **增强**：包括高斯平滑、尖峰注入、非线性变换、混合（mixup）等，提升多样性。

### 2.3 模型架构（4.5M参数）
包含五个阶段：
1. **Tokenization**：每个时间序列先归一化（sinh⁻¹变换），再分成非重叠patch（大小16），线性投影到128维。
2. **标签条件化**：对训练序列的patch表示添加学习的类别嵌入，查询序列不加。
3. **编码器堆栈**：
   - 先添加K个CLS token（K=4），通过Lt层时序注意力（RoPE）独立处理每序列；再通过Lc层跨序列注意力，让每序列条件于整个带标签上下文；最后压缩层将每序列的patch信息聚合到CLS token。
4. **上下文推理**：使用2个Transformer层在512维空间中处理所有序列的压缩表示，查询只能关注支持集和自身，不能关注其他查询。
5. **解码器**：对查询的CLS表示通过LayerNorm和两层MLP输出类别logits。

### 2.4 训练与推理
- **预训练**：在7M合成TSC任务（源自1.5M唯一VARX生成数据集经增强）上训练30K步，单张H200 GPU约40小时。优化器：矩阵参数用Muon，其余用AdamW。

- **多类扩展**：预训练最多支持10类，超过10类时采用一对多（OvR）分解。
