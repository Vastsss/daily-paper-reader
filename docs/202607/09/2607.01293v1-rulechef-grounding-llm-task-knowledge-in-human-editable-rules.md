## 摘要
我们提出RuleChef框架，该框架利用大型语言模型（LLM）为文本分类、命名实体识别（NER）或关系抽取等NLP任务生成可执行规则。规则基于任务描述和一组标注示例生成，然后根据额外的示例和人类对现有规则的反馈进行迭代改进。RuleChef还可用于通过观察给定任务中任意现有模型的输入-输出对来引导规则生成。LLM仅在训练阶段使用，用于合成规则并根据在独立验证集上测量到的失败进行迭代修补。该过程的结果是一个快速、确定且可检查的规则系统。我们在分类和NER任务上进行了初步评估。我们在Apache 2.0许可证下开源RuleChef。

## Abstract
We present RuleChef, a framework that uses large language models (LLMs) to generate executable rules for NLP tasks such as text classification, Named Entity Recognition (NER), or relation extraction. Rules are generated based on a task description and a set of labeled examples, then they are iteratively improved based both on additional examples and on human feedback overexisting rules. RuleChef can also be used to bootstrap rules using the observed input-output pairs from any existing model for a given task. LLMs are used only at learning time, synthesizing rules and iteratively patching them based on failures measured on a held-out split. The result of this process is a fast, deterministic, and inspectable rule system. Preliminary evaluation is performed on both classification and NER tasks. ### 1. 论文的核心问题与整体含义（研究动机和背景）
- **问题**：早期 NLP 系统依赖手工编写的规则，可解释性强但人力成本高、难以维护；现代基于 LLM 的系统性能优越但推理成本高、缺乏透明度，在医疗、法律、金融等敏感领域难以审计和认证。
- **动机**：希望结合规则系统的可解释性、高效率与 LLM 的强大知识能力，将 LLM 用于自动学习规则，而推理时仅运行规则，兼顾性能与透明度。
- **整体含义**：提出一种将 LLM 任务知识转化为可人工编辑、可审计的规则系统的方法，实现学习与推理分离，降低推理成本，增强可解释性。

### 2. 论文提出的方法论
- **核心思想**：使用 LLM 根据任务描述和带标签示例生成初始可执行规则（支持正则表达式、spaCy 模式、Python 代码），然后在 held-out 验证集上评估，聚类失败模式，引导 LLM 迭代修补规则；同时支持人类反馈直接修改规则或提供自然语言指导意见。
- **关键流程**：
  1. **规则合成**：为每个类别生成独立提示（包含正例、反例、任务描述、格式要求），LLM 输出规则集。
  2. **迭代精炼**：在训练集上运行当前规则，收集失败样本，聚类失败模式（如漏标、误标），将失败样本和当前规则指标输入 LLM 生成修补规则；仅当修补后开发集 F1 不下降或精度提升时接受新规则。
  3. **冲突解决与剪枝**：高优先级规则优先；精度受 Wilson 下限折扣；leave-one-out 剔除边际贡献为负的规则。
  4. **观察模式**：从已有 LLM 的输入-输出对中学习规则，无需人工标注；达到精度阈值的查询可替换 LLM 调用。
- **智能体协调**：可选的智能体协调器驱动迭代，包含规则审查（critic）、审计（auditor）、学习触发决策等子模块。
